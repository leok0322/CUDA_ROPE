================================================================================
RoPE（旋转位置编码）算法详解
================================================================================


一、背景与动机
--------------------------------------------------------------------------------

Transformer 模型本身对序列中 token 的顺序无感知（self-attention 是置换不变的），
必须额外引入位置编码才能让模型区分"第 1 个词"和"第 5 个词"。

什么是置换不变（Permutation Invariant）
------------------------------------------------------------------------

置换：将序列中的 token 重新排列顺序。例如：

    原始顺序：["猫", "吃", "鱼"]   位置 0, 1, 2
    置换后：  ["鱼", "猫", "吃"]   位置 0, 1, 2

置换不变的意思：不管输入 token 以什么顺序排列，
self-attention 计算出的每对 token 之间的注意力分数完全相同。

原因：self-attention 的计算公式为

    Q = X · W_Q
    K = X · W_K
    V = X · W_V
    score(i, j) = Q_i · K_j^T / sqrt(d)

      公式中 d 的含义：

          d 是 Q、K 向量的维度长度（head dimension），
          不是序列长度（sequence length）。

          Q_i ∈ R^d，K_j ∈ R^d，两者均为 d 维向量，
          Q_i · K_j^T 是 d 个分量的乘积之和。

      为什么除以 sqrt(d)，而不是除以序列长度：

          假设 Q 和 K 的每个分量独立同分布，均值为 0、方差为 1：

              Q_i · K_j^T = q_1*k_1 + q_2*k_2 + ... + q_d*k_d

              每一项 q_k*k_k 的期望 = 0，方差 = 1
              d 项之和的方差 = d × 1 = d
              d 项之和的标准差 = sqrt(d)

          当 d 较大时（如 64、128），点积结果的数量级会随 d 增大
          而增大，量级约为 sqrt(d)。如果不做缩放，点积值过大会导致：

              softmax(score) 的梯度极小（进入饱和区），
              反向传播时梯度接近 0，训练困难。

          除以 sqrt(d) 后，点积的标准差归一化回 1，
          softmax 的输入保持在合理范围，梯度正常流动。

          直觉理解：
              序列长度决定有多少对 (i,j) 需要计算，
              但每对的点积大小只与向量维度 d 有关，
              所以缩放因子取 sqrt(d)，与序列长度无关。


其中 score(i, j) 只取决于 token i 和 token j 各自的内容（向量值），
与它们在序列中处于第几个位置无关。

具体例子：

    句子："猫 吃 鱼"

    不加位置编码时，无论顺序如何排列，
    "猫"和"鱼"之间的注意力分数永远相同：

        score("猫", "鱼") ── 原始顺序  ──► 某个数值 X
        score("猫", "鱼") ── 打乱顺序  ──► 仍然是 X

    模型无法区分"猫在鱼前面"还是"鱼在猫前面"，
    因为两种情况下计算出的分数完全一致。

类比理解：

    self-attention 像一个"圆桌会议"，所有 token 两两互相交流，
    但桌子是圆的，没有"第一个座位"和"最后一个座位"之分。
    谁坐在谁旁边（顺序）完全不影响每对人之间交流的内容。

    位置编码的作用，就是给每个座位贴上编号，
    让模型知道"这个 token 坐在第 3 个位置，那个在第 7 个位置"。

常见位置编码方案对比：

  方案                代表工作              缺点
  ------------------  --------------------  ------------------------------------
  绝对正弦编码        原始 Transformer      外推性差，无法建模相对关系
  可学习绝对编码      BERT、GPT-2           长度固定，泛化到更长序列困难
  相对位置编码        T5, Transformer-XL   增加额外偏置项，实现复杂
  旋转位置编码 RoPE   LLaMA, Qwen, PaLM    本文重点

------------------------------------------------------------------------
1. 绝对正弦编码（Sinusoidal Position Encoding）
------------------------------------------------------------------------

代表：原始 Transformer（Vaswani et al., 2017）

数学原理：

  对序列位置 pos、embedding 维度中第 i 对维度，编码值为：

      PE(pos, 2i)   = sin( pos / 10000^(2i/d) )
      PE(pos, 2i+1) = cos( pos / 10000^(2i/d) )

  每个位置 pos 得到一个 d 维向量，低维用高频正弦，高维用低频正弦，
  形成多尺度的位置表示。不同位置的编码向量之间线性无关，可区分。

编码阶段：

  在输入层，将位置编码向量直接加到 token embedding 上，
  之后不再做任何位置相关的操作：

      输入 = Token Embedding + PE(pos)   ← 加法融合，一次性注入

      Token Embedding ──┐
                        ├──(+)──► 进入 Transformer 所有层
      PE(pos)      ──────┘

特点：
  - 固定不可学习，无需训练参数
  - 位置信息在进入第一层前就已注入，后续层无法感知"原始位置"
  - 对训练长度之外的位置，编码值可以计算，但模型未见过，外推效果差

------------------------------------------------------------------------
2. 可学习绝对编码（Learned Position Embedding）
------------------------------------------------------------------------

代表：BERT、GPT-2

数学原理：

  维护一个可训练的位置嵌入矩阵 E：

      E ∈ R^(max_len × d)

  位置 pos 的编码直接查表取第 pos 行：

      PE(pos) = E[pos]            ← 查表，每个位置独立学习一个向量

  E 在训练过程中通过反向传播更新，最终每一行学到该位置的最优表示。

编码阶段：

  与正弦编码完全相同，在输入层一次性加到 token embedding 上：

      输入 = Token Embedding + E[pos]   ← 进入所有 Transformer 层前完成

特点：
  - 灵活，位置表示由数据驱动学习
  - 最大长度 max_len 在训练时固定，超出则无对应行，无法处理更长序列
  - 不同位置之间无显式的数学关系约束，纯靠数据学习相对关系

------------------------------------------------------------------------
3. 相对位置编码（Relative Position Encoding）
------------------------------------------------------------------------

代表：T5（Raffel et al., 2020）、Transformer-XL（Dai et al., 2019）

数学原理（以 T5 为例）：

  不直接编码绝对位置，而是在注意力分数上加一个只依赖
  位置差 (i-j) 的可学习偏置 b：

      score(i, j) = (Q_i · K_j^T) / sqrt(d)  +  b(i - j)
                    ─────────────────────────    ─────────
                         内容分数                  位置偏置


  b(i-j) 是一个可学习的标量，对不同的相对距离学习不同的偏置值。
  T5 将相对距离分桶（bucket），远距离位置共享同一个偏置。

编码阶段：

  在每个 Transformer 层的 attention score 计算中注入，
  而不是在输入层：

      每一层 Self-Attention:
          score(i,j) = Q_i·K_j^T / sqrt(d)  +  b(i-j)
                                               ↑
                                      在此处加入位置信息

  与绝对编码的本质区别：
      绝对编码  ── 输入层一次注入，之后每层都看不到"原始位置"
      相对编码  ── 每层 attention 都重新注入相对位置关系

特点：
  - 天然建模相对位置，无需知道绝对坐标
  - 对未见过的序列长度有一定外推能力（远距离分桶）
  - 每层 attention 需要额外计算和存储 b(i-j) 偏置，开销为 O(n²)
  - 实现比绝对编码复杂，需修改 attention 内部计算

------------------------------------------------------------------------
4. 旋转位置编码 RoPE（本文重点，详见后续各节）
------------------------------------------------------------------------

代表：LLaMA、Qwen、PaLM、ChatGLM

编码阶段：

  在每个 Transformer 层的 attention 中，对 Q 和 K 施加旋转变换，
  位置信息通过旋转角度隐式编码到内积结果中：

      每一层 Self-Attention:
          Q_tilde = RoPE(Q, pos_m)    ← 旋转 Q
          K_tilde = RoPE(K, pos_n)    ← 旋转 K
          score   = Q_tilde · K_tilde^T / sqrt(d)

核心优势（对比上述方案）：

  绝对正弦/可学习  ── 输入层加法注入，位置与内容耦合不够自然
  相对位置（T5）   ── 显式偏置项，O(n²) 额外开销
  RoPE             ── 旋转乘法注入，内积天然只含相对位置，
                       无额外参数，外推性好（可通过缩放扩展）

RoPE 由苏剑林（Su et al., 2021）在论文
"RoFormer: Enhanced Transformer with Rotary Position Embedding"
（arXiv:2104.09864）中提出。


二、核心目标
--------------------------------------------------------------------------------

设 Query 向量 q 在位置 m，Key 向量 k 在位置 n，
我们希望设计一个位置编码函数 f，使得：

    <f(q, m), f(k, n)>  =  g(q, k, m-n)

即内积结果只依赖相对距离 m-n，而不依赖绝对位置 m 和 n。

这正是 RoPE 的设计目标——用旋转矩阵注入位置信息，
同时保持注意力分数只感知相对位置。


三、二维情形的数学推导
--------------------------------------------------------------------------------

3.1 从复数视角出发

将二维向量 (x1, x2) 视作复数 z = x1 + i·x2。

对位置 m 处的向量施加旋转角 m*theta：

    f(z, m) = z · e^(i·m·theta)
            = (x1 + i·x2)(cos(m*theta) + i·sin(m*theta))
            = (x1*cos(m*theta) - x2*sin(m*theta))
            + i(x1*sin(m*theta) + x2*cos(m*theta))

3.2 验证相对位置性质

目标：证明 <f(q,m), f(k,n)> 只依赖相对距离 (m-n)，与绝对位置 m、n 无关。

────────────────────────────────────────
前置定理 1：复数点积恒等式
────────────────────────────────────────

将二维向量视作复数，则其点积等于复数乘积的实部：

    <q, k> = Re[ q · conj(k) ]

推导：设 q = q1 + i·q2，k = k1 + i·k2

    q · conj(k) = (q1 + i·q2)(k1 - i·k2)
                = q1*k1 + q2*k2            ← 实部，正好等于点积
                + i*(q2*k1 - q1*k2)        ← 虚部，被 Re[] 丢弃

    因此 Re[ q · conj(k) ] = q1*k1 + q2*k2 = <q, k>   ✓

注：必须取共轭，若直接算 q·k，实部为 q1*k1 - q2*k2（有负号），
    与点积定义不符。

────────────────────────────────────────
前置定理 2：共轭的乘法分配律
────────────────────────────────────────

    conj(A · B) = conj(A) · conj(B)

推导：设 A = a + i·b，B = c + i·d

    左边（先乘再取共轭）：
    A · B = (ac - bd) + i·(ad + bc)
    conj(A · B) = (ac - bd) - i·(ad + bc)

    右边（各自取共轭再乘）：
    conj(A) · conj(B) = (a - i·b)(c - i·d)
                      = (ac - bd) - i·(ad + bc)

    左边 = 右边   ✓

────────────────────────────────────────
前置定理 3：单位复数取共轭等于指数取负
────────────────────────────────────────

    conj(e^(i*phi)) = e^(-i*phi)

推导：由欧拉公式 e^(i*phi) = cos(phi) + i·sin(phi)

    左边取共轭（虚部取反）：
    conj(e^(i*phi)) = cos(phi) - i·sin(phi)              ... (1)

    右边将 phi 换成 -phi 代入欧拉公式：
    e^(-i*phi) = cos(-phi) + i·sin(-phi)
               = cos(phi)  - i·sin(phi)    ← cos 偶函数，sin 奇函数

                                                          ... (2)
    (1) = (2)   ✓

几何含义：e^(i*phi) 是单位圆上辐角为 phi 的点，
取共轭 = 关于实轴镜像 = 辐角从 +phi 变为 -phi，模长不变。

────────────────────────────────────────
正式推导：逐步展开
────────────────────────────────────────

Step 1  应用复数点积恒等式（前置定理 1）
─────────────────────────────────────────
    <f(q, m), f(k, n)>
    = Re[ f(q,m) · conj(f(k,n)) ]

Step 2  代入 RoPE 编码定义 f(x, pos) = x · e^(i*pos*theta)
─────────────────────────────────────────
    = Re[ q·e^(i*m*theta) · conj(k·e^(i*n*theta)) ]

Step 3  拆开共轭（前置定理 2 + 3）
─────────────────────────────────────────
    对 conj(k·e^(i*n*theta)) 应用乘法分配律：

    conj(k · e^(i*n*theta))
    = conj(k) · conj(e^(i*n*theta))    ← 定理 2
    = conj(k) · e^(-i*n*theta)         ← 定理 3

    代回：
    = Re[ q·e^(i*m*theta) · conj(k) · e^(-i*n*theta) ]

Step 4  合并指数（指数律 e^a · e^b = e^(a+b)）
─────────────────────────────────────────
    e^(i*m*theta) · e^(-i*n*theta) = e^(i*(m-n)*theta)

    代入：
    = Re[ q · conj(k) · e^(i*(m-n)*theta) ]

Step 5  确认结果只含相对位置 (m-n)
─────────────────────────────────────────
    设 q · conj(k) = A + i·B（仅由 q 和 k 决定，与位置无关），
    展开 e^(i*(m-n)*theta) = cos((m-n)*theta) + i·sin((m-n)*theta)：

    (A + i·B) · (cos((m-n)*theta) + i·sin((m-n)*theta))
    = A·cos((m-n)*theta) - B·sin((m-n)*theta)
      ─────────────────────────────────────────   ← 实部，只含 (m-n)
    + i·(...)                                     ← 虚部，被 Re[] 丢弃

    Re[ q · conj(k) · e^(i*(m-n)*theta) ]
    = A·cos((m-n)*theta) - B·sin((m-n)*theta)
    = g(q, k, m-n)   ✓

────────────────────────────────────────
完整推导串联
────────────────────────────────────────

    <f(q,m), f(k,n)>

    Step 1  ── 复数点积恒等式
    = Re[ f(q,m) · conj(f(k,n)) ]

    Step 2  ── 代入 f(x,pos) = x · e^(i*pos*theta)
    = Re[ q·e^(i*m*theta) · conj(k·e^(i*n*theta)) ]

    Step 3  ── conj(A·B)=conj(A)·conj(B)，conj(e^(i*phi))=e^(-i*phi)
    = Re[ q·e^(i*m*theta) · conj(k) · e^(-i*n*theta) ]

    Step 4  ── 指数律 e^a · e^b = e^(a+b)
    = Re[ q · conj(k) · e^(i*(m-n)*theta) ]

    Step 5  ── 将 q·conj(k) 写成极坐标形式
    设 q·conj(k) = |q|·|k|·e^(iα)，其中
        |q|·|k| = 两向量模长之积（与位置无关）
        α       = q 与 k 的原始夹角（与位置无关）

    = Re[ |q|·|k| · e^(iα) · e^(i*(m-n)*theta) ]
    = Re[ |q|·|k| · e^(i*(α + (m-n)*theta)) ]     ── 指数相加
    = |q|·|k| · cos( α + (m-n)*theta )             ── Re[r·e^(iφ)] = r·cos(φ)

    Step 6  ── 括号内只含 (m-n)，绝对位置 m、n 完全消失
    = g(q, k, m-n)   ✓

    其中 α 和 |q|·|k| 只依赖向量内容，(m-n)*theta 是唯一的位置相关项，
    内积关于位置的变化规律为 cos((m-n)*theta) 的振荡。

────────────────────────────────────────
几何含义
────────────────────────────────────────

e^(i*m*theta) 将 q 旋转 m*theta 角，
e^(i*n*theta) 将 k 旋转 n*theta 角。

两个旋转后的向量做内积，等价于：
    将 k 反向旋转 n*theta（conj 取负角），
    再与已旋转 m*theta 的 q 做内积，
    净旋转角度 = m*theta - n*theta = (m-n)*theta。

绝对旋转量 m*theta 和 n*theta 在内积中相互抵消，
只有角度差 (m-n)*theta 留下来。
这就是 RoPE 天然建模相对位置的几何根源。

内积结果仅依赖 m-n，目标成立。


四、高维推广：分组旋转
--------------------------------------------------------------------------------

4.0 为什么需要从二维推广到高维

三、节的推导在二维（单个复数）情形下成立。
但实际 Q/K 向量的维度 d 通常为 64、128 等，远大于 2。

直接将整个 d 维向量视作一个复数是不可行的（复数只有实部和虚部两个分量）。

解决方案：将 d 维向量拆分为 d/2 个独立的二维子空间，
对每个子空间单独应用二维 RoPE，各子空间使用不同旋转频率。

这就是"分组旋转"的含义：
    d 维向量  →  d/2 组，每组 2 个相邻分量
    每组独立旋转，频率各不相同
    所有组的旋转结果拼接，得到编码后的完整向量

4.1 分组方式：相邻两维配对

设 q = [q0, q1, q2, q3, q4, q5, …, q_{d-2}, q_{d-1}]

按相邻两维分成 d/2 组：

    第 0 组：(q0,  q1 )   ← 视作一个二维向量，用 theta_0 旋转
    第 1 组：(q2,  q3 )   ← 视作一个二维向量，用 theta_1 旋转
    第 2 组：(q4,  q5 )   ← 视作一个二维向量，用 theta_2 旋转
    ...
    第 i 组：(q_{2i}, q_{2i+1})   ← 用 theta_i 旋转

每组完全独立，互不影响，各自应用三、节的二维 RoPE 公式。

4.2 旋转角频率定义

对第 i 组（i = 0, 1, …, d/2 - 1），旋转频率为：

    theta_i = base^(-2i/d)      （base 通常取 10000）

i 越小，theta_i 越大（高频，旋转快）；
i 越大，theta_i 越小（低频，旋转慢）。

不同组使用不同频率，使得整个向量的不同维度对能感知不同尺度的位置差异
（类似傅里叶分析中用不同频率的正弦波叠加表示复杂信号）。

4.3 旋转矩阵形式

对位置 m，每组单独旋转，组合成分块对角矩阵 R(m)：

    R(m) =
    [ cos(m*theta_0)  -sin(m*theta_0)                                      ]
    [ sin(m*theta_0)   cos(m*theta_0)                                      ]
    [                                  cos(m*theta_1)  -sin(m*theta_1)     ]
    [                                  sin(m*theta_1)   cos(m*theta_1)     ]
    [                                                          ...         ]
    [                                            cos(m*theta_{d/2-1})  ... ]

    分块对角的含义：每个 2×2 的小矩阵负责旋转一组维度，
    不同组之间没有任何数值耦合，完全独立。

编码后的 Query / Key：

    q_tilde = R(m) · q      ← 对每组分别做二维旋转
    k_tilde = R(n) · k

q_tilde = R(m) · q 图示
────────────────────────────────

整体流程：拆分 → 各组独立旋转 → 拼接

    原始向量 q（d 维）
    ┌────┬────┬────┬────┬────┬────┬─────┬──────┐
    │ q0 │ q1 │ q2 │ q3 │ q4 │ q5 │ … │q_{d-1}│
    └────┴────┴────┴────┴────┴────┴─────┴──────┘
      └───┬───┘   └───┬───┘   └───┬───┘    └───┬───┘
        第0组        第1组        第2组      第d/2-1组
          │            │            │             │
    旋转m*theta_0  旋转m*theta_1  旋转m*theta_2  旋转m*theta_{d/2-1}
          │            │            │             │
      ┌───┴───┐    ┌───┴───┐    ┌───┴───┐    ┌───┴───┐
      │q0'│q1'│    │q2'│q3'│    │q4'│q5'│    │ …     │
      └───┴───┘    └───┴───┘    └───┴───┘    └───────┘
          └────────────┴────────────┴──────────────┘
                               │ 拼接
    ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬──────┐
    │ q0' │ q1' │ q2' │ q3' │ q4' │ q5' │ … │q_{d-1}'│
    └─────┴─────┴─────┴─────┴─────┴─────┴─────┴──────┘
    编码后向量 q_tilde（d 维，维度数不变）

每组的旋转计算（以第 i 组为例）：

    输入：(q_{2i},  q_{2i+1})
    旋转角：φ = m * theta_i

推导：旋转矩阵从何而来
────────────────────────────────────────────
    把二维向量视作复数：

        z = q_{2i} + i · q_{2i+1}

    乘以旋转因子 e^(iφ) = cos(φ) + i·sin(φ)：

        z' = z · e^(iφ)
           = (q_{2i} + i·q_{2i+1}) · (cos(φ) + i·sin(φ))

    按复数乘法展开（实部虚部分别收集）：

        实部：q_{2i}·cos(φ)  -  q_{2i+1}·sin(φ)
        虚部：q_{2i}·sin(φ)  +  q_{2i+1}·cos(φ)

    写成矩阵形式（实部 → 第一行，虚部 → 第二行）：

    [q_{2i}' ]   [ cos(φ)  -sin(φ) ] [q_{2i}  ]
    [q_{2i+1}'] = [ sin(φ)   cos(φ) ] [q_{2i+1}]

    将 φ = m * theta_i 代入：

    [q_{2i}' ]   [ cos(m*theta_i)  -sin(m*theta_i) ] [q_{2i}  ]
    [q_{2i+1}'] = [ sin(m*theta_i)   cos(m*theta_i) ] [q_{2i+1}]

    结论：旋转矩阵不是凭空定义的，
    它就是"复数乘法展开后，实部和虚部分量的系数排列"。

    展开结果：
        q_{2i}'   = q_{2i} * cos(m*theta_i) - q_{2i+1} * sin(m*theta_i)
        q_{2i+1}' = q_{2i} * sin(m*theta_i) + q_{2i+1} * cos(m*theta_i)

每组的几何含义（复平面上的旋转）：

    把 (q_{2i}, q_{2i+1}) 视作复平面上的点，
    乘以 e^(i*m*theta_i) 将其逆时针旋转 m*theta_i 角：

            虚轴
              |
              |    · (q_{2i}', q_{2i+1}')   旋转后
              |   /
              |  /  } m*theta_i（旋转角）
              | /   }
              · (q_{2i}, q_{2i+1})          旋转前
              |/
    ──────────O──────────── 实轴

    模长不变：|(q_{2i}', q_{2i+1}')| = |(q_{2i}, q_{2i+1})|
    方向改变：辐角增加了 m*theta_i

各组频率不同，旋转角度不同：

    位置 m=3，d=6（3 组），base=10000

    theta_0 = 10000^(0/6)   = 1.000    旋转角 = 3 * 1.000 = 3.000 rad
    theta_1 = 10000^(-2/6)  = 0.046    旋转角 = 3 * 0.046 = 0.139 rad
    theta_2 = 10000^(-4/6)  = 0.002    旋转角 = 3 * 0.002 = 0.006 rad

    第 0 组旋转最多（高频，感知近距离位置差异）
    第 2 组旋转最少（低频，感知远距离位置差异）

4.4 为什么高维情形相对位置性质仍然成立

对每一组维度 i，应用三、节的二维推导：

    <f(q_{2i:2i+2}, m), f(k_{2i:2i+2}, n)> = g_i(q, k, m-n)

d/2 组的结果相加（点积是各维度乘积之和）：

    score(m,n) = q_tilde^T · k_tilde
               = 第 0 组的贡献 + 第 1 组的贡献 + … + 第 d/2-1 组的贡献
               = sum_i[ g_i(q, k, m-n) ]
               = g(q, k, m-n)    ← 每组只依赖 (m-n)，总和也只依赖 (m-n)

注意力分数推导：

    score(m, n) = q_tilde^T · k_tilde
                = q^T · R(m)^T · R(n) · k
                = q^T · R(n-m) · k

    其中 R^T(m)·R(n) = R(n-m) 的原因：
        R(m) 是正交矩阵（每列正交且模长为 1），
        正交矩阵满足 R^T = R^(-1)，
        R^T(m)·R(n) = R(-m)·R(n) = R(n-m)
        （旋转的逆 = 反向旋转，两次旋转角度相加）

4.5 逐元素高效计算（无需构造 d×d 大矩阵）

构造 d×d 矩阵再做矩阵乘法开销为 O(d²)，
利用分块对角结构，可以退化为逐元素运算，开销仅 O(d)：

对第 i 组的两个分量：

    q_tilde_{2i}   = q_{2i}   * cos(m*theta_i)  -  q_{2i+1} * sin(m*theta_i)
    q_tilde_{2i+1} = q_{2i}   * sin(m*theta_i)  +  q_{2i+1} * cos(m*theta_i)

展开验证（与二维旋转公式一致）：

    [cos  -sin] [q_{2i}  ]   [q_{2i}*cos - q_{2i+1}*sin]
    [sin   cos] [q_{2i+1}] = [q_{2i}*sin + q_{2i+1}*cos]   ✓

向量化写法（LLaMA 实现常用，避免循环）：

    cos_vec     = [cos(m*theta_0), cos(m*theta_0),
                   cos(m*theta_1), cos(m*theta_1), ...]     shape: [d]

    sin_vec     = [sin(m*theta_0), sin(m*theta_0),
                   sin(m*theta_1), sin(m*theta_1), ...]     shape: [d]

    rotate_half(q) = [-q1, q0, -q3, q2, …]
                     （每对中：奇数位取负并与偶数位互换）

    重点：
    q_tilde = q * cos_vec  +  rotate_half(q) * sin_vec

    验证第 0 组：
        q_tilde[0] = q0*cos(m*theta_0) + (-q1)*sin(m*theta_0)
                   = q0*cos - q1*sin   ✓
        q_tilde[1] = q1*cos(m*theta_0) + q0*sin(m*theta_0)
                   = q1*cos + q0*sin   ✓

Key 向量 k 同理，位置换成 n 即可。


五、频率设计：theta_i 的物理含义
--------------------------------------------------------------------------------

    theta_i = 10000^(-2i/d)

theta_i 随 i 单调递减：i=0 时 theta_0=1（最大），i=d/2-1 时 theta≈0.0001（最小）。

  维度对索引 i    theta_i 值       含义
  --------------  ---------------  ------------------------------------------
  0（低 i）       theta_0 ≈ 1      旋转快，对位置变化敏感，编码近距离精细关系
  d/2-1（高 i）   theta ≈ 1/10000  旋转慢，对位置变化不敏感，编码远距离粗粒度关系

────────────────────────────────────────
设计原因一：避免周期性混叠
────────────────────────────────────────

内积只依赖 cos(α + (m-n)·theta)（见三、节推导）。
cos 是周期函数，单一频率 theta 对应的混叠周期为 2π/theta：

    当相对距离 (m-n) 相差 2π/theta 的整数倍时，
    cos 值完全相同 → 模型无法区分这两个距离。

    示例（theta=1，仅用第 0 组）：
        混叠周期 = 2π/1 ≈ 6.28

        距离  1     → cos(1 × 1)    = cos(1)      ≈  0.540
        距离  7.28  → cos(7.28 × 1) = cos(1 + 2π) ≈  0.540  ← 与距离 1 完全相同

        距离  3     → cos(3 × 1)    = cos(3)      ≈ -0.990
        距离  9.28  → cos(9.28 × 1) = cos(3 + 2π) ≈ -0.990  ← 与距离 3 完全相同

        任意距离 x 与距离 x + 6.28，cos 值完全相等，模型无法区分。

使用多个不同频率，每组混叠周期各不相同：

    第 0 组  theta_0 = 1.000   混叠周期 ≈     6
    第 1 组  theta_1 = 0.046   混叠周期 ≈   136
    第 2 组  theta_2 = 0.002   混叠周期 ≈ 3141

两个距离要让模型无法区分，必须在所有组同时混叠。
由于各组周期互不整除，组合后的"重复周期"远超实际序列长度，
任意两个相对距离均可唯一区分。

────────────────────────────────────────
设计原因二：高频在前、低频在后的排列惯例
────────────────────────────────────────

此排列直接沿用了原始 Transformer 正弦位置编码的惯例：

    PE(pos, 2i) = sin( pos / 10000^(2i/d) )
                         ─────────────────
                         频率 = 1/10000^(2i/d)，i 越小频率越高

RoPE 令 theta_i = 10000^(-2i/d) 与之对应，前面的维度对频率最高。

实际上高频放前还是放后对模型表达能力没有本质影响
（只是维度编号与频率的对应关系不同），但统一惯例有两点好处：

  1. 与正弦编码对齐，便于理论分析和代码复用。
  2. 实现中 cos_vec / sin_vec 按维度顺序生成，
     高频在前使得向量化代码结构与数学公式一一对应，
     不需要额外的维度重排。

这与傅里叶多尺度分析思想一致：
  - 低维度对：高频旋转 → 精细编码近距离相对位置
  - 高维度对：低频旋转 → 粗粒度编码远距离相对位置

举例说明（d=6，3 组，base=10000）：

由三、节完整推导得：

    <f(q,m), f(k,n)> = |q|·|k|·cos( α + (m-n)·theta_i )
                       ───────   ─  ────────────────────
                       模长之积  固定相位  位置相关项（随 m-n 变化）

其中 α 由向量内容决定，与位置无关；
位置信息完全编码在 cos((m-n)·theta_i) 中。
不同的 (m-n) 只要让 cos 值不同，模型就能区分。

    theta_0 = 1.000   混叠周期 ≈    6   （高频，秒针）
    theta_1 = 0.046   混叠周期 ≈  136   （中频，分针）
    theta_2 = 0.002   混叠周期 ≈ 3141   （低频，时针）

  相对距离很小（m-n = 1，相邻 token）：

    组  旋转角 = 1×theta_i   cos 值          结论
    0   1 × 1.000 = 1.000   cos(1.000)=0.540  ← 明显偏离 1，可分辨
    1   1 × 0.046 = 0.046   cos(0.046)=0.999  ← 几乎不动，难以分辨
    2   1 × 0.002 = 0.002   cos(0.002)=1.000  ← 几乎不动，难以分辨

    → 近距离主要由高频组（第 0 组）负责区分。

  相对距离很大（m-n = 1000，远距离 token）：

    组  旋转角 = 1000×theta_i   cos 值              结论
    0   1000 × 1.000 = 1000    cos(1000)=0.562   ← 已绕 159 圈，混叠严重
    1   1000 × 0.046 =   46    cos(46)  ≈-0.432  ← 转了约 7 圈，仍有信息
    2   1000 × 0.002 =    2    cos(2)   =-0.416  ← 仅转了 2 rad，信息丰富

    → 远距离主要由低频组（第 2 组）负责区分。

类比时钟三根指针：

    秒针（高频）：精确区分"差几秒"的两个时刻，但无法区分"差1小时"
    分针（中频）：区分"差几分钟"的两个时刻
    时针（低频）：区分"差几小时"的两个时刻，对"差几秒"无感知

    三针合看：12 小时内任意两个时刻均可唯一区分。

RoPE 各频率组的分工与此完全一致：高频覆盖近距离精度，低频覆盖远距离范围，
所有组叠加，整个序列长度内的任意相对距离均可被唯一表示。


六、在大模型中的位置：应用于哪一层？
--------------------------------------------------------------------------------

    输入 Token Embeddings
            |
            v
    +-----------------------------------------------+
    |          Transformer Layer × N                |
    |                                               |
    |   +---------------------------------------+   |
    |   |          Self-Attention               |   |
    |   |                                       |   |
    |   |   X  -->  Linear  -->  Q, K, V        |   |
    |   |                        |   |          |   |
    |   |               RoPE ----+   |  <== 此处|   |
    |   |            编码 Q 和 K     |          |   |
    |   |                        v   v          |   |
    |   |        score = (Q_tilde · K_tilde^T) / sqrt(d)  |   |
    |   |        attn  = softmax(score) · V     |   |
    |   +---------------------------------------+   |
    |                    |                          |
    |              Add & LayerNorm                  |
    |              FFN (MLP)                        |
    |              Add & LayerNorm                  |
    +-----------------------------------------------+
            |
            v
         输出 Logits

关键点：
  1. RoPE 只作用于 Q 和 K，不作用于 V
  2. 在每个 Transformer block 的 Multi-Head Self-Attention 中注入
  3. 时机：QKV projection 之后、attention score 计算之前
  4. 每一层的每个注意力头都独立执行 RoPE（共享同一套 theta_i 参数）


七、RoPE 的作用与优势
--------------------------------------------------------------------------------

7.1 核心作用

  作用              说明
  ----------------  ----------------------------------------------------------
  注入位置信息      使模型感知 token 在序列中的顺序
  建模相对位置      注意力分数天然只依赖位置差 m-n，无绝对位置泄漏
  长度外推          训练长度之外的位置仍可计算，支持上下文窗口扩展

7.2 对比绝对位置编码

  绝对编码：
    score(m,n) = (q + PE_m)^T (k + PE_n)
    展开后含 q^T*PE_n、PE_m^T*k 等交叉项 → 隐式编码相对位置，不纯粹

  RoPE：
    score(m,n) = q^T R(n-m) k
    显式且干净地只含相对位置 (n-m)，无冗余项

7.3 上下文长度扩展技术

通过修改 base 或对 theta_i 进行缩放，可扩展有效上下文窗口：

  技术               原理
  -----------------  ---------------------------------------------------------
  Linear Scaling     将位置 m 替换为 m/s（s = 扩展倍数），旋转整体变慢
  NTK-aware Scaling  修改 base -> base * s^(d/(d-2))，统一扩展所有频率
  YaRN               对不同频率的 theta_i 分组处理：低频组插值，高频组保持原样

这些方法均基于 RoPE 的旋转结构，是 LLaMA-2-long、Qwen-long、ChatGLM
等超长上下文模型的核心技术。


八、与其他位置编码的完整对比
--------------------------------------------------------------------------------

  方案                  相对位置    外推能力  计算开销  代表模型
  --------------------  ----------  --------  --------  ----------------------
  Sinusoidal（绝对）    隐式        差        O(1)      原版 Transformer
  Learned PE（绝对）    无          很差      O(1)      BERT, GPT-2
  Relative PE（T5）     显式        中        O(n^2)    T5
  ALiBi                 显式线性    好        O(1)      BLOOM
  RoPE                  显式旋转    好        O(n*d)    LLaMA, PaLM, Qwen


九、完整数学公式汇总
--------------------------------------------------------------------------------

    定义
    -------------------------
    d        : head dimension（偶数）
    base     : 频率基数，默认 10000
    m, n     : token 位置索引（0-based）
    theta_i  : base^(-2i/d),   i = 0, 1, …, d/2 - 1

    编码
    -------------------------
    对位置 m 的向量 q 的第 i 对维度：

    [ q_tilde_{2i}   ]   [ cos(m*theta_i)  -sin(m*theta_i) ] [ q_{2i}   ]
    [ q_tilde_{2i+1} ] = [ sin(m*theta_i)   cos(m*theta_i) ] [ q_{2i+1} ]

    Key 向量 k 同理，使用位置 n。

    注意力分数
    -------------------------
    score(m, n) = q_tilde^T · k_tilde / sqrt(d)
                = q^T · R(n-m) · k / sqrt(d)

    其中 R(n-m) = R(n)^T · R(m) 是只依赖相对位置的旋转矩阵。
