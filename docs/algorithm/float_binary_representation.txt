================================================================================
浮点数基础：十进制 ⇄ 二进制 转换 + 二进制规格化表示（IEEE 754）
================================================================================

背景：build_cos_sin_cache 的 cos/sin cache 必须用 float32 计算、最后才 cast 到模型
dtype（fp16/bf16）。原因是低精度浮点在「大整数 position」「除不尽的小数 inv_freq」上会
丢精度甚至坏正确性。要理解这件事，先要理解浮点数怎么存、为什么尾数位数决定精度。
本文整理：整数、纯小数、混合数的进制互转，以及二进制规格化表示。

关联：
  src/app/util.py  build_cos_sin_cache（为何中间量恒用 float32）
  docs/algorithm/ROPE_principle.txt





================================================================================
二、十进制整数 → 二进制：除 2 取余，逆序读
================================================================================
方法：
  1) 不断 ÷2，记下每次的【余数】(0/1)；
  2) 商继续除，直到商为 0；
  3) 余数【从下往上(逆序)】读出 = 二进制。
  （第一次除得到的余数是最低位 2^0，最后一次是最高位。）

例 1：13
  13 ÷ 2 = 6  余 1   ← 最低位 2^0
   6 ÷ 2 = 3  余 0          2^1
   3 ÷ 2 = 1  余 1          2^2
   1 ÷ 2 = 0  余 1   ← 最高位 2^3
  逆序读：1101 → 13 = 1101₂
  验证：1×8 + 1×4 + 0×2 + 1×1 = 13 ✓

例 2：10
  10 ÷ 2 = 5 余 0
   5 ÷ 2 = 2 余 1
   2 ÷ 2 = 1 余 0
   1 ÷ 2 = 0 余 1
  逆序读：1010 → 10 = 1010₂  （8+0+2+0=10 ✓）

例 3：2049（呼应 fp16 的坑）
  连续 ÷2 … 最高位落在 2^11，最低位 2^0：
  2049 = 100000000001₂  （= 2^11 + 2^0 = 2048 + 1）


================================================================================
三、十进制纯小数 → 二进制：乘 2 取整，顺序读
================================================================================
方法（与整数相反）：
  1) 小数部分反复 ×2，每次取【整数部分】(0/1) 作为下一位；
  2) 余下的小数继续 ×2，直到小数部分=0（终止）或出现循环；
  3) 取到的整数位【顺序(从上往下)】读出。

例 1：0.625（能除尽）
  0.625 × 2 = 1.25 → 1  (留 0.25)
  0.25  × 2 = 0.5  → 0
  0.5   × 2 = 1.0  → 1  (留 0 → 结束)
  顺序读：101 → 0.625 = 0.101₂
  验证：2^-1 + 2^-3 = 0.5 + 0.125 = 0.625 ✓
  （能除尽：0.625 = 5/8，分母是 2 的幂。）

例 2：0.1（除不尽 → 无限循环）
  0.1 × 2 = 0.2 → 0
  0.2 × 2 = 0.4 → 0
  0.4 × 2 = 0.8 → 0
  0.8 × 2 = 1.6 → 1  (留 0.6)
  0.6 × 2 = 1.2 → 1  (留 0.2)
  0.2 × 2 = 0.4 → 0  ← 0.2 重现，开始循环
  …
  0.1 = 0.0001100110011001100…₂ = 0.0(0011) 无限循环
  ★ 根因：0.1 = 1/10，分母含质因子 5，二进制只能精确表示分母为 2 的幂的小数。
    （类比 1/3 在十进制是 0.333…）这就是 Python 里 0.1 + 0.2 ≠ 0.3 的原因。


================================================================================
四、十进制小数点位置：负幂权重
================================================================================
  位置:   …  1   1   0   1   .   1     0     1   …
  权重:      2³  2²  2¹  2⁰     2⁻¹   2⁻²   2⁻³
                              =0.5  =0.25  =0.125

  小数点左边：2^0, 2^1, 2^2 …（往左越大）
  小数点右边：2^-1, 2^-2, 2^-3 …（往右越小）

  读二进制小数 = 每位 × 其权重再相加：
    1.101₂ = 1×2^0 + 1×2^-1 + 0×2^-2 + 1×2^-3
           = 1 + 0.5 + 0 + 0.125 = 1.625
  再乘指数还原：1.101₂ × 2³ = 1.625 × 8 = 13


================================================================================
五、整数+小数混合数 → 二进制：两部分分开转再拼
================================================================================
整数部分用「除 2 取余」，小数部分用「乘 2 取整」，再拼成 xxxx.yyyy₂。

例：13.625
  整数 13    = 1101₂      （第二节）
  小数 0.625 = 0.101₂     （第三节）
  拼接：13.625 = 1101.101₂
  验证：8 + 4 + 1 + 0.5 + 0.125 = 13.625 ✓


================================================================================
六、二进制 → 规格化表示 1.fffff × 2^e（移动小数点）
================================================================================
规则：小数点每【左移 1 位】数值 ÷2，须同时 ×2^1 补偿(数值不变)；
      移到使点左边只剩一个 1 为止，移了几位指数 e 就是几。
      （类比十进制 1101. = 110.1×10¹ = … 左移补 ×10。）

例：13.625 = 1101.101₂
  1101.101₂
  = 110.1101₂ × 2^1
  = 11.01101₂ × 2^2
  = 1.101101₂ × 2^3     ← 点左边只剩一个 1，停
  → 13.625 = 1.101101₂ × 2³   （e=3，fraction=101101）

  存储时：指数 e=3；隐含位 1 不存；fraction=101101，后面补 0 到尾数位数
         （fp32 补到 23 位 / fp16 补到 10 位）。

  ★ 两个字段补 0 的方向【相反】——尾数右补、指数左补，别搞混：
    (1) fraction(尾数) = 【小数】，有效位在左、低位在右 → 在【右边(末尾)】补 0。
        末尾补 0 不改值(加的是 2⁻ᵏ=0)：0.101 = 0.1010000。
        例 13.625：fraction=101101 → fp32 存 10110100000000000000000(右补到 23 位)。
    (2) exponent(指数) = 【整数 e+bias】，不是直接存 e，要存 e+bias 后的【定宽整数】，
        在【左边(前导)】补 0，【绝不能右补】：
          整数 5=101₂：左补 00000101=5 ✓；右补 10100000=160 ✗(末尾补0=左移、乘2^k，值爆改)。
        例 13.625：fp32 exponent=e+bias=3+127=130=10000010₂(正好 8 位、无需补)；
                   fp16 exponent=3+15=18=10010₂(正好 5 位)。
        需补的例：最小正规数 2⁻¹²⁶(e=−126,fp32)→ e+bias=1 → 左补成 00000001(非 10000000=128)。
    直觉：都"在不重要的那端补 0"——小数不重要端在右(权重小)故右补；整数不重要端在左(高位空着
          即 0)故左补。
    ⟹ 完整 bit 串(13.625, fp32) = 0 | 10000010 | 10110100000000000000000
                                  sign  exp(整数,左对齐)  fraction(小数,右补0)


================================================================================
七、整数的规格化（更简单：e = 最高位 1 的位号）
================================================================================
★ 先澄清：这里规格化的是【整数值 → 浮点】，不是 int 类型本身！
  "整数"有两个意思：(a) int/uint【类型】——用补码/纯二进制存，【没有规格化】(见七点六/七点七)；
  (b) 整数【值】(如 13、2049)——恰好没小数部分的数，既可存进 int、也可存进 float。
  本节讲的是 (b) 存进 float：同一个值 13，存成 int 和 float 是两种完全不同的位模式：
    13 当 int32 存  → 00000000 00000000 00000000 00001101   （补码，无 1.fffff）
    13 当 float32 存 → 0 10000010 10100000000000000000000     （规格化 1.101₂×2³）
  关心"整数值装进浮点"正是 RoPE 的场景：positions=arange(max_position) 是整数值却放进
  【浮点张量】→ 要用本节判断"该浮点类型能否精确表示 2049 这种整数"(见第八节、第十节)。

整数末尾有个"隐藏小数点"(1101 = 1101.)，规格化本质 = 把最高位的 1 挪到个位(2^0)。
所以 e 直接 = 最高位 1 原来所在的位号，不用真数移了几位。

  ★ "位号" = 该位的 2 的幂次：从右往左，最右(个位)第 0 位(2⁰)，依次 1、2、3…
      1   1   0   1   ← 13 = 1101₂
    位号3 位号2 位号1 位号0    最高位 1 在【位号 3】
  ★ "e=最高位 1 的位号" 与 "数移了几位" 是同一个数(故可直接读位号、省去移动)：
    方式A(数次数,第六节)：1101.→110.1→11.01→1.101，移 3 次 ⟹ e=3
    方式B(读位号,本节)  ：最高位 1 在位号 3 ⟹ 直接 e=3
    二者必相等：小数点从最右(位号0 之后)移到最高位 1 右边(位号 k 之后)，移动格数 = k = 该位号。
  ★ 等价速算：e = (二进制总位数 − 1) = floor(log₂N)。
      例 13="1101"共 4 位 → e=4−1=3；2048="100…0"共 12 位 → e=12−1=11。

  13   = 1101₂          (最高位在 2³) → 1.101₂ × 2³        e=3, fraction=101
  10   = 1010₂          (最高位在 2³) → 1.010₂ × 2³        e=3, fraction=010
  2048 = 100000000000₂  (最高位在 2¹¹)→ 1.0₂ × 2¹¹          e=11, fraction=000…0(全 0，精确)
  2049 = 100000000001₂  (2¹¹ 与 2⁰) → 1.00000000001₂ × 2¹¹  e=11, fraction 需 11 位(末位=原 2⁰)
    ★ 2049/2048 = 1+2⁻¹¹ → 那个 1 落在第 11 个 fraction 位；fp16 尾数仅 10 位、装不下第 11 位
      → 2049 是 fp16【第一个】不能精确表示的整数(舍入到 2048)。
      对比 2050 = 1.0000000001₂×2¹¹(1 在第 10 位、需 10 位)→ 可精确；故 [2048,4096) 只存
      偶数 2048,2050,2052…(步长 2=2^(11-10))，奇数全丢。详见第八节判据。

================================================================================
一、浮点数的三段结构（IEEE 754）
================================================================================
  value = (-1)^sign × 1.fffff(二进制) × 2^(exponent - bias)
          └─符号─┘   └── 尾数 ──┘       └──── 指数 ────┘
  bit 布局：  [ sign ][ exponent ][ fraction ]
               1位       n位          m位

  格式      总位  符号  指数(n)  尾数(m,存储)  有效精度(含隐含1)  指数bias
  float32   32    1     8        23            24                 127
  fp16(half)16    1     5        10            11                 15
  bf16      16    1     8        7             8                  127

★ bf16 与 float32 指数位相同(8) → 表示范围一样(不易溢出)；只是尾数 23→7，精度大降。
  这是 "brain float" 的取舍：深度学习更怕溢出(范围)、不那么怕精度。
  fp16 则是范围小、精度居中。

-------------------------------------------------------------------- (1) sign 符号位
- sign 是最高位【单独 1 个 bit】，只取 0/1。(-1)^sign 把它变成正负号：
    sign=0 → (-1)^0 = +1（正数）；  sign=1 → (-1)^1 = -1（负数）
- 除 sign 外 exponent/fraction 都一样 → 正负只差这 1 个 bit（故有 +0 与 -0 两个零）。

-------------------------------------------------------------------- (2) 尾数 fraction
- 1.fffff = 规格化尾数：点左边永远是 1（隐含位，不存储），fffff 才是真正存的位。
- 整个 1.fffff ∈ [1, 2)；指数 2^e 把它放大/缩小到正确量级。
- 有效精度 = 存储尾数位 m + 1（隐含位）。
  ★ 为什么是 m+1：有效精度看"值里有几位二进制数字"，不是"存了几位"。
    1.f₁f₂…f_m 一共 (1 个点左 1) + (m 个 fraction) = m+1 位，这 m+1 位都决定数值。
    - 点左的 1 为何能不存：规格化要求 1.fffff∈[1,2)，挪完小数点后点左那位【只可能是 1】
      (是 0 就还能左移=没规格化；二进制单位最大就是 1)。既然恒为 1、无悬念 → IEEE 不存它
      (叫隐含位/hidden bit)，用时硬件自动补回 → 只花 m 位存储、却得 m+1 位精度(白赚一位)。
    - 例 float32(m=23)：13.625 = 1.10110100000000000000000₂×2³，存的 fraction 23 位，
      实际有效数字 24 位 → 故"能分多少档"用 2^(m+1)=2^24≈1.68e7(~7 位十进制)，非 2^23。
    - 例外：次正规数(subnormal，exponent 全 0)隐含位改为 0、尾数变 0.fffff，无这免费的 1，
      精度退回 m 位。"m+1"是针对正常数(normalized)而言。
  ★ "有效精度 m+1" 是【有效数字位数(significant figures)】，不是【小数点后位数】！
    浮点 = 尾数 × 2^e 恰好分工：尾数(1.fffff,m+1 位)定"有几位有效数字(精度/形状)"；
    指数(2^e)定"小数点放哪(量级/位置)"。故 m+1 只数有效数字个数，与小数点位置无关。
    - 同 7 位十进制有效数字的 float32，小数点位数可千差万别：
        1234567.0(点后 0 位)、12.34567(点后 5 位)、0.0001234567(点后 10 位)
      —— 有效数字都是 7(都信 1234567)，但"点后位数"从 0 到 10。有效精度恒定(尾数定)、
      点后位数随量级变(指数定)。
    - 二进制同理：1.101101₂×2³=1101.101₂(点后3位)、×2⁻²=0.01101101₂(点后8位)，
      同一尾数 7 位有效数字不变、小数点位置全变。m+1 数的是这个 7，不是点后位数。
    - 这也是 machine epsilon=2⁻ᵐ 恒定的原因：它衡量"有效数字"(与量级无关)故为常数；
      而绝对误差(≈小数点能精确到第几位)随量级变(见第八节)。

-------------------------------------------------------------------- (3) 指数与 bias
- 指数 e 可正(大数)可负(小数)，但 exponent 字段只能存非负整数(0 ~ 2^n-1)。
  解法：存 e+bias(保证非负)、用时减回 → 公式里的 (exponent - bias)：
      存储 exponent 字段 = 真实指数 e + bias
      真实指数 e         = exponent 字段 - bias
- bias = 2^(n-1) - 1：≈指数字段范围 [0, 2^n-1] 的整数中点，让 e 近似对称覆盖正负。
    float32: 2^7-1=127；fp16: 2^4-1=15；bf16: 2^7-1=127(指数位同 float32)。
- 为什么减 1 / 不用满范围：exponent 全 0、全 1 两个码被【保留】：
    全 0 (=0)      → 表示 0 与 次正规数(subnormal)
    全 1 (=2^n-1)  → 表示 ±∞ 与 NaN
  故正常数的存储值只用 1 ~ 2^n-2。代入 e=存储值-bias：
    float32: 存 1~254 → e ∈ [-126, +127]，约 2^-126~2^128 ≈ 1.2e-38 ~ 3.4e38
    fp16   : 存 1~30  → e ∈ [-14, +15]，  约 2^-14~2^16   ≈ 6e-5 ~ 65504
  ★ 大端为何是 2^128(而非 2^127，明明 e_max=127)：数值 = 尾数 × 2^e，最大值要尾数也取最大。
    尾数最大 = 1.111…1₂ ≈ 2(准确 2-2⁻²³，无限接近 2)→ 最大值 ≈ 2 × 2^127 = 2^(127+1) = 2^128
    (≈3.4e38，无限接近达不到)。小端则尾数取最小 1.0、无此 ×2 → 就是 2^(e_min)=2^-126。
    即：大端 = 尾数≈2 × 2^(e_max) = 2^(e_max+1)；小端 = 1.0 × 2^(e_min) = 2^(e_min)。
    (fp16 同理：大端 ≈2×2^15 = 2^16 = 65536，实际 65504；小端 2^-14。)
  取 2^(n-1)-1(而非 2^(n-1))让范围略偏正：正侧多一格、负侧最小端再由 subnormal 向下延伸。
  ★ 为何【下取整】(bias=127)而非上取整(bias=128)：中点 127.5 非整数，必须二选一；两种
    宽度一样，只差偏正/偏负：
      bias=127(下取整)：e=存储值-127 → e∈[-126,+127]  ← 偏正，e_max 更大，最大值≈2^128
      bias=128(上取整)：e=存储值-128 → e∈[-127,+126]  ← 偏负，e_max 只到 +126，最大值≈2^127
    IEEE 选下取整(偏正)，把多出的一格给【大数端】，因为：
      (1) 小数端已被次正规数(subnormal，exponent 全 0)渐进延伸(float32 可到 ~2^-149)，不缺
          这格负指数；而大数端是【硬墙】——超 e_max 直接溢出成 ±∞，没有渐进上溢，更需要这格。
      (2) 保证正规数倒数不溢出：最小正规数 2^-126 的倒数 2^126 < 最大值 2^128，稳稳在范围内；
          若 bias=128 则 2^-127 倒数 2^127≈最大值，余量更紧。
    故"向下取整"不是随意，而是把宝贵的一格留给更需要的大端(硬溢出墙 + 倒数 headroom)。
- 用 bias(移码)而非补码：位模式可直接当无符号整数比大小，硬件比较更简单。

★ 指数正负管【量级(大/小)】，不管【正负(>0/<0)】——别和 sign 混：
    指数 e 为正、大 → 2^e 大 → 数【大】          1.0 × 2^10 = 1024
    指数 e = 0       → 2^0 = 1 → 数在 1 附近       1.5 × 2^0 = 1.5
    指数 e 为负      → 2^e < 1 → 数【小于 1】      1.6 × 2^-4 = 0.1
    指数 e 很负      → 2^e 极小 → 趋近 0           1.0 × 2^-126 ≈ 1.2e-38
  ⟹ 浮点指数取【负数】是为了表示【绝对值小于 1 的小数】(0~1 之间乃至趋近 0)，
     【不是】为了表示小于 0 的负数！数的正负只由 sign 位决定，与指数正负无关。
  例：0.1 = 1.1001…₂ × 2⁻⁴，指数 −4 但 0.1 是正数(sign=0)。四种组合任意搭：
    sign=0,e>0 → +大数(+13)  sign=1,e>0 → −大数(−13)
    sign=0,e<0 → +小数(+0.1) sign=1,e<0 → −小数(−0.1)
    

================================================================================
七点五、浮点数三段结构的完整编码举例（十进制 ⇄ 二进制 bit）
================================================================================
把一个十进制数走完整链路：转二进制 → 规格化 → 填 sign/exponent/fraction 三段 → 拼成
bit 串；再反向解回十进制验证。

────────────────────────── 例 1：+13.625 编成 float32（32 bit）
十进制 → 二进制 → 规格化（前面各节）：
  13.625 = 1101.101₂ = 1.101101₂ × 2³        （e=3，尾数 1.101101）
三段：
  sign     = 0                       （正数）
  exponent = e + bias = 3 + 127 = 130 = 10000010₂   （8 位）
  fraction = 101101，右补 0 到 23 位 = 10110100000000000000000
拼成 32 bit：
  0 10000010 10110100000000000000000
  └s┘└─exp──┘└────────frac(23)────────┘
反向解回十进制（验证）：
  sign=0 → +
  真实指数 = 130 - 127 = 3
  尾数 1.101101₂ = 1 + 2⁻¹+2⁻³+2⁻⁴+2⁻⁶ = 1 + 0.5+0.125+0.0625+0.015625 = 1.703125
  value = +1 × 1.703125 × 2³ = 1.703125 × 8 = 13.625 ✓

────────────────────────── 例 2：-13.625（只翻 sign 位）
  1 10000010 10110100000000000000000     （sign 改 1，其余完全相同）
  value = (-1)^1 × 1.703125 × 2³ = -13.625 ✓

────────────────────────── 例 3：同一个 +13.625 编成 fp16（16 bit）
  规格化不变：1.101101₂ × 2³
  sign     = 0
  exponent = 3 + 15(bias) = 18 = 10010₂     （5 位）
  fraction = 101101，右补 0 到 10 位 = 1011010000
  拼成 16 bit：0 10010 1011010000
  解回：指数 18-15=3，尾数 1.703125 → 1.703125 × 8 = 13.625 ✓（fp16 也能精确表示它）

────────────────────────── 例 4：+13 编成 float32（整数，e=最高位位号）
  13 = 1101₂ = 1.101₂ × 2³
  sign=0；exponent=3+127=130=10000010₂；fraction=101 补 0 = 10100000000000000000000
  → 0 10000010 10100000000000000000000
  解回：1.101₂=1.625，×2³=13 ✓

────────────────────────── 例 5：负指数小数 0.1（除不尽 → 截断有误差）
  0.1 = 1.1001100110011…₂ × 2⁻⁴          （尾数 1.(1001) 无限循环）
  float32：sign=0；exponent = -4 + 127 = 123 = 01111011₂；
           fraction 取前 23 位（循环被截断）→ 存的值 ≈ 0.100000001490116…（非精确）
  fp16   ：exponent = -4 + 15 = 11 = 01011₂；fraction 取前 10 位 1001100110（第11位0舍去）
           → 1.1001100110₂ × 2⁻⁴ = 0.0999755859375（详见第九节）
  ★ 对比例 1/4：13、13.625 二进制有限 → 可精确编码；0.1 二进制无限循环 → 必有舍入误差。


================================================================================
七点六、int / uint / float：三种编码对比 + 为何浮点恒有符号
================================================================================
整数和浮点是【两套完全不同的编码】。整数没有指数位/尾数位，不存在 1.fffff×2^e 结构。

  float32 :  [ sign 1 ][ exponent 8 ][ fraction 23 ]   ← 有指数、有尾数（规格化）
  uint32  :  [          纯二进制 32 位的大小           ]  ← 无符号、无指数（直接位权和）
  int32   :  [ 32 位补码（最高位是符号，但用补码规则） ]  ← 有符号、无指数

-------------------------------------------------------------------- uint（无符号整数）
  N 位全部表示大小，纯二进制按位权求和；无符号位 → 表示不了负数。
    uint8 的 13 = 00001101₂ = 8+4+1 = 13
    范围：0 ~ 2^N − 1   （uint8: 0~255；uint32: 0~4294967295）
  （PyTorch schema 用 int 不用 uint，部分即因 uint 无法表示 −1。）

-------------------------------------------------------------------- int（有符号整数）
  最高位是符号位，但用【补码】而非浮点那种 (-1)^sign×大小。详见七点七。
    范围：−2^(N−1) ~ 2^(N−1)−1   （int8: −128~127；int32: −2147483648~2147483647）

-------------------------------------------------------------------- 为何浮点【恒有符号】、只有整数才分有/无符号
  主流标准/硬件(IEEE 754、C/C++、CUDA、PyTorch、NumPy)里：
    - 浮点：float16/32/64、bf16、fp8 全部【有符号】，没有 "unsigned float"。
    - 整数：int 与 uint 两种都有。
  原因——"无符号"的唯一好处是把符号位改作大小、翻倍正范围；这对整数值钱、对浮点没用：
    (1) 整数每一位都直接计入大小 → 省下符号位正范围翻倍(uint8 127→255)，且索引/长度/
        字节/位掩码天然非负，值得。
    (2) 浮点范围由【指数】决定(float32 ±3.4e38 来自 8 位指数)，砍掉 sign 位只多 1 bit、
        几乎无收益，却再也表示不了负数——而权重/梯度/RoPE 的 −x2 到处是负数，完全不划算。
  补充：新出的 fp8(E4M3/E5M2)仍有 sign 位；图像 unorm8 看似"无符号小数"实为 uint÷255，
        不是浮点类型；torch 浮点 dtype 全有符号，无无符号浮点 dtype。


================================================================================
七点七、整数补码（two's complement）详解：为何"取反 +1"= 取负
================================================================================
int 最高位虽是符号位，但【不是】(-1)^sign×大小(那叫原码)，而是【补码】。

-------------------------------------------------------------------- 求负：逐位取反，再 +1
  int8 例：
    +13   : 00001101
    取反~ : 11110010    （每位 0↔1 翻转，即按位 NOT）
    +1    : 11110011    = −13

-------------------------------------------------------------------- 为什么"取反 +1"= 取负（推导）
  关键：x 与其取反 ~x 逐位互补，相加每位都是 1：
      00001101 (x=13) + 11110010 (~x) = 11111111 (全 1) = 2^N − 1
  ⟹  x + ~x = 2^N − 1
  ⟹  ~x = 2^N − 1 − x
  ⟹  ~x + 1 = 2^N − x
  而整数运算自带"模 2^N 回绕"(进位溢出丢弃)，在 mod 2^N 下：
      2^N − x ≡ −x        （因 (2^N − x) + x = 2^N，进位丢掉后 = 0）
  所以 ~x+1 = 2^N − x 就代表 −x。代入：~13+1 = 256−13 = 243 = 11110011₂ = −13。

  验证 −13 + 13 = 0（普通二进制加法）：
      11110011 (−13) + 00001101 (+13) = 1 00000000（9 位）
      第 9 位进位丢弃 → 00000000 = 0  ✓   （正负相加得 0 = 相反数的定义）

-------------------------------------------------------------------- 等价读法：最高位权重取负
  补码 = "最高位(MSB)权重为 −2^(N−1)，其余位照常为正"：
    int8 各位权重：  -2⁷  2⁶  2⁵  2⁴  2³  2²  2¹  2⁰
                    =-128  64  32  16   8   4   2   1
    11110011 = −128 + 64+32+16 + 2+1 = −128 + 115 = −13  ✓
  "取反+1"是【求补码的算法】，"最高位负权重"是【补码的读法】，二者一致。

-------------------------------------------------------------------- 自反性
  "取反+1"自反：对 x 得 −x，对 −x 再做得回 x。
    11110011(−13) 取反→00001100 +1→00001101 = +13  ✓

-------------------------------------------------------------------- 为何整数选补码（好处）
  (1) 加减统一：a−b 直接当 a+(−b)，正负共用一套加法器，无需按符号特判。
  (2) 0 唯一：只有 00000000 一个零（对比浮点/原码有 +0、−0 两个）。
  (3) 范围偏负一格：int8 −128~+127；−128=10000000(仅 MSB=−2⁷)，但 +128 越界 → 负数多一个。
  对照浮点：浮点本就要对阶/规格化、较复杂，用直观的独立 sign 位反而方便，故不用补码。


================================================================================
八、为什么尾数位数决定精度（步长 & 整数可表示上界）
================================================================================
一个数 = 1.fffff × 2^e。固定指数 e 下，尾数最低位变化 1 单位 → 数值变化：
    步长(ulp) = 2^e × 2^(-尾数位) = 2^(e - 尾数位)        （m = 尾数位数）
  指数 e 越大(数越大) → 步长越大 → 越稀疏。

-------------------------------------------------------------------- 步长公式怎么来的（分两步）
(1) 尾数最低位的权重 = 2⁻ᵐ：尾数 1.f₁f₂…f_m 的小数位权重依次 2⁻¹,2⁻²,…,2⁻ᵐ，
    最右那位(f_m)权重 = 2⁻ᵐ。故"尾数最低位变 1 单位"→ 1.fffff 这个数只变 2⁻ᵐ。
      fp16(m=10) 2⁻¹⁰≈9.8e-4；bf16(m=7) 2⁻⁷≈7.8e-3；float32(m=23) 2⁻²³≈1.2e-7
    这是尾数能分辨的"最小刻度"。
(2) 再乘公共倍率 2^e：真正数值 = 1.fffff × 2^e，尾数变 2⁻ᵐ → 数值变 2⁻ᵐ×2^e = 2^(e−m)。
    指数 2^e 把那把"刻度 2⁻ᵐ 的尺子"整体放大 2^e 倍 → e 越大、刻度越粗。

-------------------------------------------------------------------- 最直观框架：每个二进制区间被等分成固定 2^m 份
把数轴按 2 的幂切成区间(binade) [2^e, 2^(e+1))：
  - 区间宽度 = 2^(e+1) − 2^e = 2^e；
  - 区间内所有数同用一个 e、尾数 1.fffff 有 2^m 种取值 → 被等分成 2^m 个点；
  - 份数恒为 2^m(与区间大小无关)。
  ⟹ 步长 = 区间宽度 / 份数 = 2^e / 2^m = 2^(e−m)。
  ★ 直觉：份数固定、区间随 e 翻倍变宽 → 数越大、同样多的点摊得越开 → 越稀疏、步长越大。

-------------------------------------------------------------------- 具体数字（取尾数 m=2 位，手算）
尾数 1.ff₂ 有 4 种：1.00,1.01,1.10,1.11 = 1.0,1.25,1.5,1.75（区间内步长 0.25=2⁻²）。
每升一档指数(×2)整组翻倍、步长也翻倍：
  e=0 (×1)：  1.0   1.25  1.5   1.75     步长 0.25 = 2^(0-2)
  e=1 (×2)：  2.0   2.5   3.0   3.5      步长 0.5  = 2^(1-2)
  e=2 (×4)：  4     5     6     7        步长 1    = 2^(2-2)
  e=3 (×8)：  8     10    12    14       步长 2    = 2^(3-2)   ← 9,11,13 表示不了！
  e=4 (×16)： 16    20    24    28       步长 4    = 2^(4-2)
  数轴（越往右越疏）：
  1  1.25 1.5 1.75  2   2.5  3  3.5  4    5    6    7  8      10     12   14
  |--|--|--|------|----|----|--|----|----|----|----|--|------|------|----|→
  └ step .25 ┘     └─ step .5 ─┘    └─ step 1 ─┘      └──── step 2 ────┘
    (e=0,密)         (e=1)            (e=2)              (e=3,疏)
  看 e=3 行：奇数 9/11/13 全没了(步长=2) → 这正是"大整数丢精度"：e≥m 时步长≥1 跳过整数。

-------------------------------------------------------------------- 推论：相对误差恒定（machine epsilon 的由来）
  相对步长 = 步长 / 数值 ≈ 2^(e−m) / 2^e = 2⁻ᵐ，与 e 无关
  → 浮点【相对】误差恒定(~2⁻ᵐ)=machine epsilon；【绝对】误差则随数值变大而变大。
  这就是浮点"大数粗、小数细"非均匀刻度的根因：公共倍率 2^e 放大/缩小固定尾数刻度 2⁻ᵐ。

整数 N 落在 [2^e, 2^(e+1)) 时写成 1.fffff × 2^e，步长 = 2^(e-m)，m=尾数位（有效精度=m+1）：
  - e ≤ m：步长 ≤ 1 → 该区间每个整数都精确；
  - e = m+1：步长 = 2 → 只能存偶数；e=m+2 步长=4 …
  ⇒ 连续整数全精确的上界 = 2^(m+1)。

  格式    尾数位 m   连续整数精确到 2^(m+1)   之后步长
  float32 23         2^24 = 16,777,216        >16M 间隔2，>32M 间隔4…
  fp16    10         2^11 = 2048              >2048 间隔2，>4096 间隔4…
  bf16    7          2^8  = 256               >256 间隔2，>512 间隔4…

判据（整数 N 规格化为 1.fffff × 2^e 后）：
  e ≤ 尾数位数 → fraction 放得下 → 精确；
  e > 尾数位数 → 低位的 1 被挤出 fraction → 被舍入。
  （fp16 尾数10 → N≤2^11=2048 全精确；bf16 尾数7 → N≤2^8=256；fp32 尾数23 → N≤2^24=16M。）
  ★ 含端点：上界 2^(m+1) 本身是 2 的幂(1.0×2^(m+1)、0 个 fraction 位)→【也精确】；故精确集是
    【闭区间 [0, 2^(m+1)]】，第一个 gap 在【2^(m+1)+1】(fp16 即 2048 精确、2049 第一个丢)。
    2^(m+1) 是边界点：既是连续段最后一个(精确)，又是下一档(步长 2)的起点。

  ★ 为何阈值正好 2^(m+1)：一条对应链（e=m 是"最大还安全"的指数）：
      e=m  ⟺  N∈[2^m, 2^(m+1))  ⟺  最坏(奇数)需 m 个 fraction 位  ⟺  format 的 m 位刚好装下
    (1) e=最高位 1 的位号(第七节)→ e=m 即 N∈[2^m, 2^(m+1))。
    (2) N=1.fffff×2^m，fraction 第 k 位 ↔ 整数位号 m−k；要够到个位(位号 0)需第 m 个 fraction 位。
        奇数(个位=1)是最坏情况 → 需满 m 位。
    (3) format 恰好 m 位 → 最坏的奇数也塞得下 → 整个 [2^m,2^(m+1)) 全精确。
    再上一档 e=m+1：最坏奇数个位映射到第 m+1 位 > m 个槽 → 装不下、舍入。故 e=m 是临界。
    验证(fp16,m=10)：区间 [1024,2048) 最坏奇数 2047=1.1111111111₂×2¹⁰，fraction=1111111111
      恰好 10 位 → 精确 ✓；而 e=11 的 2049 需 11 位 → 舍入(见第七节)。
    ⟹ 所有 N ≤ 2^(m+1) 全精确(闭区间，含端点 2^(m+1)=2 的幂)；第一个 gap = 2^(m+1)+1。

机器相对精度(machine epsilon) ≈ 2^(-m)（m=尾数位；= 1.0 之后那个间隙的相对值）：
  float32 2^-23 ≈ 1.2e-7（~7 位十进制有效数字）
  fp16    2^-10 ≈ 9.8e-4（~3 位）
  bf16    2^-7  ≈ 7.8e-3（~2 位）

-------------------------------------------------------------------- ⚠ 步长 ≠ 误差：步长是"网格间隔"，误差 ≤ 步长/2
常见误解："绝对步长 = 整数的绝对误差"——不对。步长是【可表示数之间的间隔(网格密度)】，
误差是【某数离最近网格点的距离】，二者相关但不相等：
  可表示数： ●--------●--------●     (● 间隔 = 步长)；真值 x 落在中间，误差=x 到最近 ● 的距离
  round-to-nearest → 最大误差 = 步长/2(x 在正中间时)；x 正好在 ● 上 → 误差 = 0。
  ⟹ 误差 ≤ 步长/2，【不是】误差 = 步长。
对整数：
  e ≤ m(步长≤1)  ：整数全精确 → 误差 = 0(不是步长!)
  e = m+1(步长2) ：奇数舍到最近偶数 → 最大误差 = 1 = 步长/2（例 fp16 2049→2048，误差 1）
  e = m+2(步长4) ：最大误差 = 2 = 步长/2
  例 bf16 10000：步长 64，舍到 9984、误差 16(<步长/2=32，因 10000 不在正中)→ 误差 16≠步长 64。
四个量别混：
  绝对步长(ULP)        = 网格间隔        = 2^(e-m)
  整数最大绝对误差     = 最坏舍入(中点)   = 步长/2  = 2^(e-m-1)
  相对步长(=ε)         = 步长/值         = 2^-m
  整数最大相对误差     = 误差/值         = ε/2    = 2^-(m+1)
小结：步长是【分辨率/粒度尺度】(误差的上界尺度)，不是某数的实际误差；可表示的数误差=0，
      不可表示的数误差 ≤ 步长/2。"步长≥1→整数开始丢"对(网格比 1 疏)，但"步长=该整数误差"错。

-------------------------------------------------------------------- 误差累积：n 步叠加 → √n·ε(随机) ~ n·ε(最坏)
★ 先界定 ε(两个约定别混)：
  · machine epsilon ε = 2^-m = 1.0 之后那个间隙(=ULP(1.0)，C 的 FLT_EPSILON / numpy finfo.eps)。
  · unit roundoff  u = ε/2 = 2^-(m+1) = round-to-nearest 的【严格】单步最大相对误差(半个 ULP)。
  ⟹ 严格的单步舍入界是 u=ε/2(与上一节"误差≤步长/2"一致)。

★ 为何累积式用 ε(=2^-m)、不用严格的 ε/2 —— 分"能不能"和"为什么这么选"：
  (一) 合法性(能这么放)：ε ≥ u=ε/2 → 用 ε 得到的是【更宽松但仍正确】的上界(n·ε ≥ n·u ≥ 真实误差)，
       绝不会低估 → 安全。
  (二) 实用性(为什么这么选)：
    1. 量级判断够用：误差分析只关心数量级("几个 ε"、">>2π?"、"要不要 fp32?")，差个常数 2 不改变结论。
    2. 书写干净：ε=2^-m 是熟知常数(FLT_EPSILON)，避免每处都带 /2。
    3. 更贴近真实：理想单次舍入才 ≤ε/2；真实多步运算(exp/log、非最近舍入、算法本身)每步常 >ε/2、
       接近甚至超过 ε → 用 ε 当每步尺度反而更接近实情，而非过度乐观。
  (三) 代价/严格化：只松了一个常数因子 2；要严格上界，把下式所有 ε 换成 u=ε/2 即可：
       √n·(ε/2) ~ n·(ε/2)。结论(随机 √n、最坏 n；大 n 必须 fp32)完全不变。

单步误差 ~ε 尺度(严格 ≤ε/2)。但一串 n 个运算/累加，每步带 ~ε 误差 eᵢ(符号有正有负)，总误差
= Σeᵢ。它有多大取决于符号会不会抵消：

(1) 最坏(全同号、完全相关) → 线性叠加 = n·ε（确定上界，绝不更糟）。

(2) 随机(独立、正负抵消) → 像"随机游走"，典型只到 √n·ε。为何：
    · 总误差 Σeᵢ 是个【随机变量】，每次跑都不同 → 要用一个数概括它的"典型幅度"。
    · 均值没用：E[Σeᵢ]=ΣE[eᵢ]≈0（正负抵消，平均无偏）——但这不代表误差=0，只是围绕 0 波动。
    · 故用【标准差】(=均方根 RMS)衡量"围绕 0 的典型偏离"；均值=0 时 标准差=RMS=典型误差大小。
    · 标准差 = √n·ε 的来历：
        方差相加(独立的关键)：Var(X+Y)=Var(X)+Var(Y)+2Cov；独立→Cov=0→Var(Σeᵢ)=ΣVar(eᵢ)=n·σ²
          (σ≈ε)。若误差相关/同向→Cov>0→交叉项不消→逼近最坏 n·ε。
        开根号(回到误差单位，方差单位是 ε²)：标准差=√(n·σ²)=√n·ε。√n 就来自"方差线性×n 再开方"。
    · 结论：误差 ~ 0 ± √n·ε —— 典型落在 ±√n·ε。

类比：最坏=n 步全朝一个方向、走 n；随机=醉汉每步随机左右、典型只走 √n（经典随机游走结论）。

具体(ε=0.8%, bf16)：
  n=4(inv_freq 几步)  ：随机 √4·ε=1.6%   最坏 4·ε=3.2%   → 都是几个 ε
  n=100               ：随机 8%          最坏 80%
  n=4096(matmul K 维) ：随机 51%         最坏 灾难        → 必须 fp32 累加(第十点五机制A)
实际多接近随机 √n·ε(舍入方向大多独立)；误差相关时逼近最坏 n·ε。工程上：√n·ε 估典型、n·ε 估上界。
共同结论：步数/累加项越多，越该升 fp32。


================================================================================
八点五、误差传播：加减→绝对误差相加；乘除→相对误差相加（含绝对↔相对换算）
================================================================================
两个带误差的数做运算，结果误差怎么算？规则【不是】"绝对相乘、相对相加"，而是：
  · 加 / 减 → 【绝对】误差相加
  · 乘 / 除 → 【相对】误差相加
（"相对×数值=绝对"是同一个数的两种误差表示之间【换算】，不是运算传播，别混。）

记号：绝对误差 Δa（存成 a+Δa）；相对误差 δa=Δa/|a|（a+ Δa存成 a(1+δa)，因为Δa=a*δa）。

-------------------------------------------------------------------- 加 / 减：绝对误差相加
  c = a + b：c_存 = (a+Δa)+(b+Δb) = (a+b) + (Δa+Δb) ⟹ Δc = Δa + Δb
  c = a − b：Δc = Δa − Δb，最坏 |Δa|+|Δb| ⟹ 绝对误差仍叠加。
  直觉：数值直接堆叠，误差(同单位的量)也直接堆叠。
  · 加减的【相对】误差(由 Δ 换算 δc=Δc/|c|，代入 Δa=a·δa)：
      加 c=a+b：δc = (a·δa + b·δb)/(a+b) = δa、δb 按 |a|、|b| 的【加权平均】
        → 同号相加时介于 δa、δb 之间，【有界、不放大】(输入 ~ε 则输出仍 ~ε)。
      减 c=a−b：δc = (a·δa − b·δb)/(a−b) → a≈b 时分母→0 → δc【暴涨】。
  ⚠ 相消(cancellation)：a≈b 时 a−b 很小，但绝对误差 Δa+Δb 不变 → 相对误差 δc 暴涨
    (分母 |c| 趋 0)。这就是 RMSNorm 方差 E[x²]−E[x]² 要 fp32 的原因(第十点五机制A)。
    ★ 关键非对称：加法相对误差有界(加权平均)，减法(相消)相对误差才暴涨。

-------------------------------------------------------------------- 乘 / 除：相对误差相加
  c = a×b：c_存 = a(1+δa)·b(1+δb) = ab(1+δa+δb+δaδb) ≈ ab(1+δa+δb)
    （δaδb 是二阶小量，忽略）⟹ δc ≈ δa + δb
  c = a/b：δc ≈ δa + δb（1/b 的相对误差 −δb，平方项忽略后绝对值仍叠加）。
  直觉：乘除是"按比例缩放"，误差以比例(百分比)方式叠加 → 相对误差相加。
  · 乘除的【绝对】误差(由 δ 换算 Δc=|c|·δc)：
      乘 c=a×b：Δc = bΔa + aΔb（直接展开 (a+Δa)(b+Δb)−ab ≈ aΔb+bΔa；
        即每个绝对误差被【另一个操作数】放大 → 操作数越大、绝对误差被放得越大）。
      除 c=a/b：Δc = |a/b|·(δa+δb) = Δa/|b| + |a|·Δb/b²。
  幂法则(特例)：c = a^p → δc ≈ p·δa（相对误差 ×p）。完整推导(两法)：
      法1(二项/泰勒展开)：a 存成 a(1+δa) →
        c_存 = (a(1+δa))^p = a^p·(1+δa)^p
        对小量用 (1+x)^p ≈ 1+px（一阶泰勒/二项近似，x=δa 很小）：
        (1+δa)^p ≈ 1+p·δa → c_存 ≈ a^p(1+p·δa) = c(1+p·δa) ⟹ δc ≈ p·δa
      法2(对数微分)：c=a^p → ln c = p·ln a → 两边微分 dc/c = p·da/a → δc = p·δa
        （dc/c=c 相对误差、da/a=a 相对误差，故"相对误差 ×p"）。
      一致性：a^p=exp(p·ln a)，与第十节崩点2b 的链路 ln→×p→exp 推出的 δ(a^p)=p·δa 相同。
      例 p=0.5(开方)：δ(√a)=0.5·δa（base 0.16% → √ 后 0.08%，见第十节验证）。

-------------------------------------------------------------------- 绝对 ↔ 相对：换算（这里才有"乘"）
  同一个数的两种误差表示互转（不是传播）：
    绝对误差 = 相对误差 × |数值|     Δ = δ × |x|
    相对误差 = 绝对误差 / |数值|     δ = Δ / |x|
  这就是"乘"的来源——别和上面运算传播的"相加"混。

-------------------------------------------------------------------- 该看绝对还是相对？(常见误解：以为"只看绝对、相对只为算绝对")
  ★ 对浮点而言【相对误差才是原生/根本】的：浮点相对精度恒定(ε)、绝对精度随量级变(=ε×值)，
    故每个浮点运算【原生给的是相对误差 ~ε】；【绝对误差是派生的】(Δ=δ×值，要乘数值才得到)。
    所以严格说是"绝对由相对算出"，不是"相对为了算绝对"。
  ★ 最终看哪个 = 看【下游容差是固定量(绝对)还是百分比(相对)】：
    看【绝对】(下游有固定量纲/门槛)：cos/sin 比 2π；阈值比较 |x−0.5|；index/position 须精确；
      加减累加。→ 需把原生相对 ε 换算成绝对(×值)再比。
    看【相对】(下游关心"几位有效数字/百分比")：有效数字；allclose 的 rtol；权重/激活/梯度
      (错 1% 就是 1%，与大小无关)；乘除传播。→ 直接用相对，不必换绝对。
    ⟹ ML 里相对误差常常【就是终点】(尺度无关)，并非只是中间量。
  ★ RoPE 为何像"相对→绝对"：那里下游门槛 2π 是【绝对量】，故流程=原生相对 ε →(×angle 换算)
    绝对 → 比 2π。这只是"门槛是绝对量"时的流程，不是普适规律(门槛是相对时就直接用相对)。
  实用判据：问"下游能容忍的误差是一个固定的量(绝对)还是一个百分比(相对)？"，据此选；两者 Δ=δ×|x| 互换。

-------------------------------------------------------------------- 对照 RoPE：传播 + 换算各用一次
  angle = pos × inv_freq（乘法）：
    ① 传播(乘→相对相加)：δ(angle) ≈ δ(pos)+δ(inv_freq) = 0+ε = ε
    ② 换算(相对×值→绝对)：Δ(angle) = δ(angle)×angle = ε×4095 ≈ 33 弧度（崩点3）
  ①的"加"是传播规则，②的"乘"是绝对↔相对换算，两者作用不同。

-------------------------------------------------------------------- 小结表（每种运算两种误差都列）
  运算       绝对误差 Δc                  相对误差 δc                  谁简洁
  加 a+b     Δa+Δb                        (a·δa+b·δb)/(a+b) 加权平均    绝对(相加)
  减 a−b     Δa−Δb                        (a·δa−b·δb)/(a−b) 相消→暴涨   绝对(相加)
  乘 a×b     bΔa+aΔb（另一操作数放大）     δa+δb                        相对(相加)
  除 a/b     Δa/|b|+|a|Δb/b²              δa+δb                        相对(相加)
  幂 a^p     |c|·p·δa                     p·δa                         相对(×p)
  换算(同一数)  Δ = δ×|x|                 δ = Δ/|x|                    —（互转，非传播）
  规律：加减→绝对误差形式简洁(相加)；乘除→相对误差形式简洁(相加)。另一种误差用 Δ=δ×|x| 换算得到。
        例外提示：减法的相对误差会因相消(a≈b)暴涨，是数值计算最大的坑。


================================================================================
九、实例：0.1 在三种精度下的存储与误差
================================================================================
0.1 = 1.1001100110011…₂ × 2⁻⁴（尾数 1.(1001) 无限循环 → 必被截断）

fp16（尾数 10 位）：取前 10 位 1001100110，第 11 位=0 → 舍去(向下)：
  存 1.1001100110₂ × 2⁻⁴
   = (1 + 2⁻¹ + 2⁻⁴ + 2⁻⁵ + 2⁻⁸ + 2⁻⁹) × 2⁻⁴
   = 1.599609375 × 2⁻⁴ = 0.0999755859375
  误差 = 0.1 − 0.0999755859375 ≈ 2.4e-5

  格式     存储的近似值                绝对误差
  bf16(7)  0.1000976…                  ≈ 9.8e-5
  fp16(10) 0.0999755859375             ≈ 2.4e-5
  float32  0.100000001490116…          ≈ 1.5e-9
  真值     0.1000000000…               0


================================================================================
十、回到 RoPE：cos/sin cache 为何必须 fp32 计算（三个崩点 + 误差传播）
================================================================================
对应代码 (src/app/util.py build_cos_sin_cache，配置 rotary_dim=64, max_position=4096,
base=10000)：
    inv_freq  = 1.0 / (base ** (arange(0, rotary_dim, 2)/rotary_dim))   # 频率 [half]
    positions = arange(max_position)                                    # 位置 0..4095
    freqs     = outer(positions, inv_freq)   # 角度 angle = pos × inv_freq
    cos, sin  = cos(freqs), sin(freqs)        # 结果 ∈ [-1,1]
    return cache.to(dtype)                    # 仅最终结果降精度
最终都汇入 angle = pos × inv_freq 再送 cos/sin。低精度在三处各崩一次。

-------------------------------------------------------------------- inv_freq 的范围
  inv_freq_k = 10000^(-k/32) = 10^(-k/8)，k=0..31：
    k=0→1，k=8→0.1，k=16→0.01，k=24→0.001，k=31→10^-3.875≈1.33e-4
  从 1 到 ~1.3e-4，跨约 4 个数量级。

-------------------------------------------------------------------- 崩点 1：positions 是大整数 → 表示不下
  positions = arange(4096) 是【大整数值】放进浮点张量 → 大整数表示问题(第八节)：
    bf16 整数精确上界 2^8=256 → 257 起被量化；4095 附近步长 2^(11-7)=16
    fp16 整数精确上界 2^11=2048 → 2049 起步长 2^(11-10)=2，奇数位置全丢
    float32 上界 2^24=16M → 4095 远在内，精确 ✓
  → bf16/fp16 下不同 token 拿到相同 position → RoPE 位置编码错乱。

-------------------------------------------------------------------- 崩点 2：inv_freq 跨量级 → 相对精度不足(非表示不下)
  注意：inv_freq 里的 arange(0,64,2)=[0..62] 是【小整数】，fp16/bf16 都存得下；inv_freq
  最小 1.3e-4 也在表示范围内。所以崩点 2【不是大整数/表示不下】，而是【有效数字不够】：
  (a) base=10000 在 bf16 存不准：底数一开始就错。完整计算（步长法）：
      ① 定 e：2^13=8192 ≤ 10000 < 16384=2^14 → e=13。
      ② bf16 m=7 → 步长 = 2^(e-m) = 2^(13-7) = 2^6 = 64。
      ③ 可表示值都是步长的整数倍（8192=128×64 本身是 64 倍数）→ 10000 精确 ⟺ 10000/64 整。
      ④ 10000/64 = 156.25（非整）→ 存不下；候选 156×64=9984、157×64=10048。
      ⑤ 比距离：|10000-9984|=16 < |10048-10000|=48 → 舍到 9984（或 156.25<.5 → 取 156）。
      ⑥ 误差 = (10000-9984)/10000 = 16/10000 = 0.16%。
      尾数法交叉验证：10000/8192 = 1.220703125；小数 0.220703125 ÷ 2^-7(=1/128) = 28.25
        非整 → 舍 28 → 28/128=0.21875 → 1.21875×8192 = 9984（同上）。
      (fp16 m=10 → 步长 2^(13-10)=8，10000/8=1250 整 → fp16 精确存 10000；此条只坑 bf16。)
  (b) 幂结果跨 4 个数量级、低精度有效位不足（相对精度问题，非表示不下）。完整推导：
      ① ε=2^-m 是相对精度(machine epsilon)：
           bf16 ε=2^-7 =1/128 =0.0078125 ≈0.78% ≈0.8%
           fp16 ε=2^-10=1/1024=0.0009766 ≈0.098% ≈0.1%
      ② 换十进制有效数字 d：能信 d 位 ⟺ 相对精度≈10^-d，令 10^-d≈ε → d≈-log₁₀ε；
         又 ε=2^-m → d = m×log₁₀2，log₁₀2≈0.301(来自 2^10≈10^3，每 10 个二进制位≈3 个十进制位)：
           bf16 d=7×0.301 =2.1 → ~2 位有效数字
           fp16 d=10×0.301=3.0 → ~3 位
      ③ 具体看"只保 ~2 位"(承②，inv_freq≈0.75，即 10^(-1/8)=0.7499)：
           绝对不确定 = 相对×值 = 0.8%×0.75 ≈ 0.006 = 6×10^-3 → 顶到【千分位】(小数点后第3位)
           值落在 0.75 ± 0.006 = [0.744, 0.756]。判据：绝对误差够到哪一位，那位及更低位失真：
             十分位 7：误差 0.006 << 0.1 → 完全稳 ✓
             百分位 5：误差 0.006 < 0.01 → 大致稳(边缘 ±1，范围 0.74~0.75)
             千分位起：范围 0.744~0.756 这位 0~9 全可能 → 纯噪声 ✗
           ⟹ 头 2 位"0.75"可信、第 3 位起是噪声(= bf16 ~2 位有效数字的含义)。
      ★ "有效位不足"与"相对误差"是【同一件事】：都在描述类型的【相对精度 ε=2^-m】，只是单位/
           视角不同——有效数字 d 是"能信几位"(位数)、相对误差 ε 是"误差占百分之几"(比例)，靠
           d=-log₁₀ε 互换。例 bf16："只信 2 位"⟺"相对误差 ~1%"是同一句话。二者都【尺度无关】
           (0.75 和 750000 都信 2 位)，故对得上"相对"误差、对不上"绝对"误差。
           ⚠ 仅适用【相对类】问题(本崩点2 inv_freq)；崩点1(positions 大整数表示不下)是【绝对/整数
             表示】问题，与有效数字/相对误差无关，别套进去(故 positions、inv_freq 用 fp32 的原因不同)。
      ④ "每个频率带 ~ε 相对误差"——误差从哪来 + 每步用【绝对】还是【相对】误差：
         记号：δ=相对误差、Δ=绝对误差，换算 Δ=δ×|值|(第八点五节)。本节统一符号：
           δ_base = base 的相对误差(存错)=0.16%；  r 是算 u=p·ln(base) 时的相对舍入 ~ε；
           p是指数 = k/32 ≤ 0.969；  |u| = p·ln(base) ≤ 8.9。
         三条传播规则(决定每步追哪种误差，见第八点五节)：
           · ln ：相对→绝对   d(ln x)=dx/x  → Δ(ln x)=δx
           · exp：绝对→相对   d(eᵘ)/eᵘ=du   → δ(eᵘ)=Δu
           · 乘常数 c：绝对×c、相对不变；   取倒数 1/x：相对不变(符号翻转)
         inv_freq = 1/base^p，base^p=exp(p·ln base)，p=k/32≤0.969。

         ── 来源A：base 本身存错(0.16%)沿链路传到 inv_freq，逐节点标误差类型 ──
           节点                 值      误差(标类型)            这步规则
           base                 9984    δ=16/10000=0.16%【相对】 浮点存储舍入(δ~ε级)
           ─ln→  ln(base)       9.21    Δ=δ_base=0.0016【绝对】  ln：相对→绝对(Δ=δ)
           ─×p→  u=p·ln(base)   ≤8.9    Δ≈p·0.0016【绝对】       乘常数 p：绝对×p
           ─exp→ base^p         ≤7480   δ≈p·0.16%【相对】        exp：绝对→相对(δ=Δu)
           ─1/x→ inv_freq       —       δ≈p·0.16%【相对】        取倒数：相对不变
           净：δ(inv_freq) ≈ p×δ_base （= 幂法则 δ(a^p)=p·δa）。
           ★ 看 ln/exp 如何交替翻转误差类型：相对(base)→绝对(ln,u)→相对(base^p,inv_freq)。

         ── 符号(先把 u 说清) ──
           u = p·ln(base) = 【喂进 exp 的那个数】(因 base^p=exp(u))；
           代数：ln(base)=ln(10000)≈9.21，p=k/32≤0.969 → u=p×9.21，范围 [0, 8.9](k=31 时最大)。
           例 k=31：u=0.969×9.21≈8.9，base^p=exp(8.9)≈7480。
           δ_base=base 相对误差(存错)=0.16%；ε=machine epsilon(bf16=0.8%)；
           r=【算 u 这一步新产生】的相对舍入(~ε)。

         ── 来源B：用【同一张节点表】描述(独立于 base 是否存错；误差从【u 节点】进入：算 u 时新舍入 r) ──
           链路每个浮点算子(ln、×p、exp、取倒数)各引入【相对】舍入 ~ε(≤ε/2，第八节)；重点是 exp 放大：
           节点                值     误差(标类型)                这步规则
           u=p·ln(base)        ≤8.9   δ=r=ε=0.8%【相对】          算 u 时新舍入(相对，~ε)  ← B 起点
             ↳ 转绝对喂 exp           Δu_B=r·|u|=0.008×8.9≈0.071  相对→绝对：×|u| ← ★放大在此(u 大就放大)
           ─exp→ base^p        ≤7480  δ=Δu_B=r·|u|≈7.1%【相对】   exp：绝对→相对(δ=Δu，1:1 不缩放)
           ─1/x→ inv_freq      —      δ=r·|u|≈7.1%【相对】        取倒数：相对不变
           净(仅 B)：δ(inv_freq)_B ≈ r·|u| ≈ 7%（放大倍数 = |u| ≈ 8.9）
           ★ 和来源A 同一张表、同一套规则；区别只是【从哪个节点进入】：A 从 base 进(经 ln→×p 到 u)，
             B 从 u 进(算 u 的新舍入)。两者到 u 都是【绝对误差】，再共用 exp(1:1)、取倒数。
           ★ "放大 ×|u|" 在【输入侧】(u 的相对 r→绝对，因 exp 只吃绝对)，u 大就放大；【不是】"exp 输出
             相对后再乘值"(那是求输出【绝对】×e^u=×7480，inv_freq 关心相对、不走)。exp 本身 1:1。
             数值核对：exp(8.9)≈7480、exp(8.9±0.071)≈7480×(1±0.071) → 7.1% ✓。

         ── 合并 A+B：两份【不同】误差，都落到"节点 u 的绝对误差 Δu"上、再一起过 exp ──
           来源       u 的绝对误差 Δu        来历
           A(搬运)    Δu_A = p·δ_base        base 已存错(9984) 经 ln(相对→绝对)、×p 传到 u
           B(新产生)  Δu_B = r·|u|           算 u 时新舍入 r，经 Δ=δ×值 成绝对
           合计       Δu = Δu_A + Δu_B = p·δ_base + r·|u|
           过 exp(绝对→相对)→ δ(base^p) = Δu；取倒数不变 → δ(inv_freq) = p·δ_base + r·|u|
           数值(bf16，k=31：p=0.969、|u|=8.9、δ_base=0.0016、r=ε=0.008)：
             Δu_A = p·δ_base = 0.969×0.0016 ≈ 0.0016
             Δu_B = r·|u|    = 0.008×8.9    ≈ 0.071
             Δu_B / Δu_A ≈ 45 → 【来源B(计算+exp 放大)主导】，A(base 存错)反而小。
           ★ 同一条链、同一条 exp 规则(δ=Δu)，但喂进的 Δu 来历不同(A=p·δ_base、B=r·|u|)→ 两份、相加。

         ── 具体验证(p=0.5=开方，验来源A的幂法则；此处只看 base 存错那份) ──
           真值：base^0.5=√10000=100，inv_freq=1/100=0.01
           bf16：base=9984 → √9984=99.92，inv_freq=1/99.92=0.010008
           相对误差 δ(inv_freq)=(0.010008−0.01)/0.01=0.08% = 0.5×0.16% = p·δ_base ✓

         净效果：δ(inv_freq) ≈ p·δ_base + r·|u|（A+B 相加，B 常主导）；bf16 ~1%~7% 量级、仅 ~2 位
           有效(与 ③ 一致)，fp16 ~0.1%。这点【相对】误差再乘大 position 经换算成巨大【绝对】角度
           误差(崩点3) → 源头(base 存储 + 幂运算)都必须 fp32。
    
      ★ 为何出现 exp/ln(误差链的根源)：非整数次幂【定义/实现】就是 exp·ln——
            a^p = (e^(ln a))^p = e^(p·ln a) = exp(p·ln a)   （恒等式：a=e^(ln a)，再幂的幂指数相乘）
          原因：① 硬件/库没有"任意次幂"指令，只有 exp、ln(或 exp2/log2)原语 →
                  pow(a,b)=Python a**b(浮点) 内部都展开成 exp(b·ln a)；
                ② 小数指数 p=k/32(如 0.03125)不能像整数次幂那样连乘 → 只能走 exp·ln。
          ⟹ base**(k/32) 必然 = exp((k/32)·ln base)，故链路含 ln、exp，才有下面的类型翻转与放大。


-------------------------------------------------------------------- 崩点 3(最致命)：相对误差 × 大 position → 绝对角度误差 >> 2π
  cos/sin 是周期函数(周期 2π≈6.28)，真正决定值的是 angle mod 2π。而：
  (1) 乘积相对误差相加：angle=pos×inv_freq → δ(angle)≈δ(pos)+δ(inv_freq)。
      pos 用 fp32(δ≈0)、inv_freq 用 bf16(δ≈ε) → angle 相对误差 ≈ ε，原封传给角度。
  (2) 绝对误差 = 相对误差 × 数值，随 angle 暴涨(相对恒定、绝对放大)：
      绝对角度误差 ≈ ε × angle。bf16：0.8% × (最大~4095) ≈ 33 弧度。
  (3) 33 弧度 / 2π ≈ 5 个周期 → 连"在第几圈、圈里哪个角度"都不知道 → cos/sin = 随机。
      fp16(2 弧度≈0.3 周期)也错得离谱；float32(1e-7×4095≈4e-4 弧度 << 2π)才几乎无误差。
  ★ 这条是"放大器"：任何进入 angle 的相对误差都被大 position 放大成 >> 2π 的绝对误差。
    要让绝对误差 << 2π，相对误差须 ≤ 2π/4095 ≈ 1.5e-3 还更小 → bf16(0.8%)/fp16(0.1%)
    都不够，唯 float32(1e-7)够。即使 positions 精确，光 inv_freq 的 ε 也能毁掉角度。

-------------------------------------------------------------------- positions 与 inv_freq 用 fp32 的原因【不同】
  positions：大整数【表示不下】(步长≥1，绝对) → fp32 让大整数精确。
  inv_freq ：小整数 arange 本能存下，问题是 base 存不准 + 幂结果【有效位不足】(相对精度) →
             fp32 让频率值够准；且这点相对误差经 ×大 position 放大为巨大角度误差。
  殊途同归：都为保证 angle=pos×inv_freq 准到 << 一个周期(2π)，cos/sin 才正确。

-------------------------------------------------------------------- 结论（HF / vLLM 标准做法）
  cache 中间量(arange / base^x / cos / sin)恒用 float32 计算；仅【最终结果】cast 到模型
  dtype：return cache.to(dtype)。关键：中间量(angle 达数千)需高【绝对】精度，输出(cos/sin
  ∈[-1,1])只需 [-1,1] 内 ~ε 相对误差、降精度安全。
  → "输出精度"听配置，"计算精度"为正确性固定 fp32。precision 花在【过程】(大角度)，不花在
    【结果】(小 cos/sin)。
  对照：rms_norm_pure 同理——x.float() 上算 mean(x^2)，再 .to(in_dtype)。

-------------------------------------------------------------------- ★ "需 fp32"指的是 cache 构造，不是旋转施加(rope_pure 可 bf16)
  RoPE 分两段，精度命门只在第一段：
    build_cos_sin_cache：positions / inv_freq / angle=pos×inv_freq / cos / sin  ← 恒 fp32(三崩点)
          │  cache.to(dtype) 降回 bf16
          ▼
    rope_pure：拿【已算好】的 cos/sin 对 x 做 2D 旋转 r=x1·c∓x2·s             ← bf16 即可
  rope_pure 在 bf16 安全，因为三机制【一个都没踩】：
    · 机制A(累加)：每个输出元素只是【2 项】组合(x1·c∓x2·s)，N=2，非大 K 累加 → 误差 ~几个 ε，有界。
    · 机制B(超越/大角度)：【这里没有 sin/cos 运算】——cos/sin 是 cache 里已算好的值，rope_pure 只
      是拿来相乘；危险的"大角度喂 sin/cos"(崩点3)已发生在 cache 构造(fp32)里。
    · 机制C(吞没)：|c|,|s|≤1、x1/x2 同量级，两项量级相当 → 无小加大被吞。
    且旋转保范数(|c|,|s|≤1 不放大)。⟹ "RoPE 需 fp32"= cos/sin cache 的计算；rope_pure 消费的是
    已在 fp32 算好、再 cast 成 bf16 的 cos/sin，自身只做有界 2 项旋转，故全程不升精度(对照
    rms_norm_pure 内部 x.float()，rope_pure【不】升精度——因为确实不需要)。


================================================================================
十点五、推广：大模型里还有哪些计算需要 fp32（混合精度 / AMP）
================================================================================
混合精度(AMP)：默认 bf16/fp16，只把【命中下面三机制之一】的算子升 fp32 算、算完降回。
RoPE cache(第十节)其实就是机制B(超越)+机制A(乘累加)的具体案例。判据始终是第八点五的
"看下游容差是绝对量还是相对量"——下面每个算子都标清楚误差是【绝对】还是【相对】、怎么算出来的。

-------------------------------------------------------------------- 机制A：累加 / 归约（误差随项数 N 放大 + 相消）
本质：把 N 个数相加(Σ)。每加一步，当前部分和 sₖ 被舍入，引入【绝对】误差 ~ε·|sₖ|；N 步的
  绝对误差按第八节累积：随机 √N·ε·|s|、最坏 N·ε·|s|。换算成结果的【相对】误差 = (总绝对)/|s|
  = 随机 √N·ε、最坏 N·ε。⟹ N 越大相对精度越差，故大 N 的求和必须用 fp32 累加器。
  ★ 注意是"绝对舍入相加→再÷|s| 得相对"，不是"相对误差相加"(相对相加是乘除规则，第八点五)。
  ★ 另一面是相消(第八点五减法)：若求和含正负抵消使 |s|→0，相对误差 = 绝对/|s| 暴涨。

  ★ 误差有【两段来源】(以乘累加 GEMM 为例)，别只记得后一段：
    (1) 输入项本身的传播误差(进求和【前】就有)：被加的每项 pₖ=aₖbₖ 是【乘积】，按乘法规则
        δ(pₖ)=δ(aₖ)+δ(bₖ)（乘→【相对】相加，第八点五；两个 bf16 因子各 ~ε，故每项 ~2ε，
        再加乘法结果自身舍入 ~ε）。这些【输入侧相对误差】换成绝对 Δpₖ=δ(pₖ)·|pₖ| 后随求和叠加。
    (2) 累加【新引入】的舍入(上面"本质"那段)：每加一步部分和被舍入 ~ε·|sₖ| → √N·ε / N·ε 的 N 因子。
    ⟹ 总误差 = 输入项误差(每项 ~2ε) + 累加 N 因子；fp32 累加器主要压住的是(2)那段会随 N 爆的部分。
    ★★ 两段在 N 上【行为相反】，别把它们乘起来当成 N·2ε：
       (1) 输入侧 2ε【不随 N 涨】：N 项相加相对误差是【加权平均】(第八点五加法)，N 项各 ~2ε 的
           加权平均仍 ~2ε——分子 Σ2ε·|pₖ| 与分母 |s|=Σ|pₖ| 同步随 N 放大、约掉，故恒 ~2ε。
       (2) 累加舍入【每步是 ε 不是 2ε】：那 2ε 是【乘法】留在 pₖ 上的；到【加法】这步只做【一次】
           舍入(相对 ~ε≤ε/2)，每加一步新引入一个 ε → √N·ε / N·ε。不因被加数是乘积就变每步 2ε。
       ⟹ 总 ≈ 2ε(不随 N) + √N·ε(随 N)；N=4096 时 1.6% + 51%，累加段完全压倒输入段。

  算子与计算过程(ε_bf16≈0.8%)：
  · GEMM/matmul c=Σ_{k=1..K} aₖbₖ：结论=【乘用 bf16、加(累加器)用 fp32】——两步误差行为相反：
      (a) 乘 aₖ·bₖ → bf16：每项 ~2ε(乘法相对相加)，逐元素、【不随 K 涨】、有界 → bf16 够。
      (b) 加 Σ(累加器) → fp32：每步新引入 ~ε，K 项累成 √K·ε / K·ε，【随 K 爆】(机制A) → 必须 fp32。
      (c) 代数字(K=4096)：累加段 bf16 最坏 K·ε≈3300%、随机 √K·ε≈51%(第八节同例)；fp32(ε≈6e-8)
          K·ε≈0.025% → 压住。乘法段一直 ~1.6%、与 K 无关。
      (d) 硬件：Tensor Core 就是这结构——输入 aₖ/bₖ bf16(省显存/带宽)、乘法低精度、【累加器内部恒
          fp32】(部分和 sₖ 全程 fp32 相加)、最后可再降回 bf16 输出。bf16 训练快又稳即源于此：
          省的是乘法/存储位宽，保的是累加精度。
  · Norm 统计 mean(x²) / 方差：
      (a) 哪步危险：D 项【求和】(mean 的核心)随 D 累积 → 机制A 本体，必须 fp32 累加。
      (b) RMSNorm vs LayerNorm：RMSNorm 只有 mean(x²)，【无减法、无相消】；LayerNorm 方差
          Var=E[x²]−E[x]² 多一个【减法】→ 相消(第八点五)，E[x²]≈E[x]² 时低精度直接崩。
      (c) x²(逐元素乘)危险吗：不危险——误差 ~2ε 有界、【不随 D 涨】。故硬要求只是"求和"
          (LayerNorm 再加"减法")，x² 本身可 bf16。
      (d) 实践：直接把 x 整个升 fp32(本项目 rms_norm_pure 的 x.float())，x²/求和一起 fp32——
          因为升一次几乎不花钱，省得纠结哪步该升。
      (e) 误差性质：求和=绝对累积→相对 √D·ε；(仅 LayerNorm)相消=相对暴涨。
  · softmax(z) = exp(zᵢ−max) / Σⱼ exp(zⱼ−max)：【同时命中 B 和 A】，故整段 fp32。
      (a) 分子 exp = 超越函数 → 机制B：exp 的相对误差 = 输入绝对误差 Δu=ε·|u|，|u| 大就不小 → 要 fp32。
      (b) 分母 Σexp = 对 N(序列长)项求和 → 机制A：N 大，累加器要 fp32。
      (c) 减 max 一举两得：①防 e^u 溢出(bf16 上界 ~3.4e38)；②缩小 |u| → 降 (a) 的 exp 误差。
      (d) 误差性质：(a) exp 输入绝对、输出相对(机制B)；(b) 求和绝对累积→相对(机制A)。
  · 梯度 all-reduce / loss / grad-norm：跨上万元素/多卡求和，N 极大 → fp32 归约。

-------------------------------------------------------------------- 机制B：超越函数 / 大动态范围（exp、log、sin/cos）
本质：exp/log/sin/cos 这类函数对输入的【绝对】误差敏感，且常把误差放大或要求大角度/大范围精确。
  · exp(u)：输出【相对】误差 δ = 输入【绝对】误差 Δu(1:1，因 d(eᵘ)/eᵘ=du)。坑在【喂进去前】要把
      u 的相对误差 r 换算成绝对 Δu=r·|u|(第八点五换算)——u 大则放大 |u| 倍(第十节崩点2b 来源B)。
      故 logits/softmax 的 exp 用 fp32，且先减 max 把 u 拉小、同时防 e^u 溢出(bf16 上界 ~3.4e38)。
  · log / 交叉熵 ln(p)：把输入【相对】误差转成输出【绝对】误差(d ln=dp/p)；p→0 时 ln(p)→−∞、
      斜率 1/p 爆炸 → 微小相对误差变巨大绝对误差 → fp32 + clamp。
  · sin/cos(RoPE)：要求【角度的绝对误差 << 2π】(第十节崩点3)。angle=pos×inv_freq 相对误差 ε，
      换算绝对 = ε×angle，position 大时 bf16 0.8%×4095≈33 弧度 >> 2π → 随机；唯 fp32 够。
  共性：下游容差是【绝对量】(2π、溢出上界、−∞)，故原生相对 ε 必须 ×值换算成绝对再比 → 大值/大范围下放大。

-------------------------------------------------------------------- 机制C：小量加大量被"吞掉"（swamping，主权重 / 优化器）
本质：a+b 中 |b| << |a| 时，b 落在 a 的步长(=ε·|a|，第八节)以下 → 加了等于没加(b 被吞)。
  判据：|b|/|a| < ε(单步) 或 < ε/2(舍到最近) ⟹ 完全丢失。这是【绝对】误差视角(步长是绝对量)。
  · 优化器主权重更新 w ← w − lr·g：典型 lr·g/|w| ~ 1e-3 ~ 1e-7。bf16(ε≈0.8%)下，更新量比步长
      还小 → w 原地不动、训练停滞。故存一份【fp32 主权重】，更新累加在 fp32 上，前向再降 bf16
      (这正是 AMP "fp32 master weights" 的由来)。误差性质：被吞=绝对量低于步长。
  · Adam 一阶/二阶矩 m、v 的滑动累加，EMA：同样是"小增量持续累加进大累加量"，低精度增量被吞 →
      m/v、EMA 用 fp32。
  ★ 与机制A 的区别：A 是"很多同量级数相加，误差随 N 涨"；C 是"两个悬殊量级相加，小的整个丢失"。

-------------------------------------------------------------------- 一句话归纳
  升 fp32 的算子＝命中三机制之一：A 累加/归约(GEMM 累加器、Norm 统计/相消、softmax 分母、loss、
  grad)；B 超越/大范围(exp、log、sin/cos、logits)；C 小加大被吞(主权重更新、Adam m/v、EMA)。
  其余(逐元素乘加、激活、bf16 GEMM 的乘法部分)留低精度。判误差类型：A 绝对累积→相对 √N·ε/N·ε
  (+相消)；B 下游绝对门槛、相对×值放大；C 绝对量低于步长被吞。


================================================================================
十一、速查
================================================================================
- 十进制整数 → 二进制：÷2 取余、逆序读。       例 13 = 1101₂
- 十进制小数 → 二进制：×2 取整、顺序读。       例 0.625 = 0.101₂；0.1 除不尽循环
- 混合数：整数/小数分开转再拼。               例 13.625 = 1101.101₂
- 位权：左 2^0,2^1…；右 2^-1,2^-2…；按权求和读值。
- 规格化 1.fffff × 2^e：左移小数点到只剩一个 1；整数则 e=最高位 1 的位号。
- 三段编码：sign(1bit,0正1负,(-1)^sign) / exponent(存 e+bias) / fraction(尾数,不含隐含1)。
    例 +13.625 → float32: 0 10000010 10110100000000000000000（exp=3+127=130）。
- bias = 2^(指数位-1)-1：float32=127/fp16=15/bf16=127；存 e+bias 让指数能表负、用时减回。
    缘由：≈指数字段中点使范围近似对称；全0/全1 两码留给 0·subnormal / ∞·NaN，故减 1。
- 指数正负管【量级】：e>0→大数、e=0→约 1、e<0→【小于 1 的小数】(如 0.1)、e 很负→趋近 0。
    指数取负是为表示【绝对值<1 的小数】，不是表示【负数】；正负由 sign 位管，与指数无关。
- 隐含位：规格化后点左的 1 不存储 → 有效精度 = 尾数位 + 1。
- 步长 = 2^(e - m)；连续整数精确上界 = 2^(m+1)（fp16 2048 / bf16 256 / fp32 16M），m=尾数位。
- 步长 ≠ 误差：步长=可表示数的间隔(网格密度)；误差=离最近网格点的距离 ≤ 步长/2(可表示则=0)。
    整数最大绝对误差=步长/2=2^(e-m-1)；最大相对误差=ε/2=2^-(m+1)。"步长=该整数误差"是误解。
- machine eps ε ≈ 2^-m(1.0 后的间隙,FLT_EPSILON)：fp32 ~1e-7 / fp16 ~1e-3 / bf16 ~8e-3。
    严格单步最大舍入误差 = unit roundoff u = ε/2 = 2^-(m+1)；累积式用 ε 当每步尺度是放宽 2× 的量级估计。
- 误差累积(n 步每步~ε)：随机(独立、正负抵消)~√n·ε(典型，标准差，因方差相加再开方)；最坏(同号)
    ~n·ε(上界)。总误差均值=0 故用标准差(=RMS)代表典型幅度。步数/累加项越多越该升 fp32。
- 误差传播(第八点五)：加减→绝对误差相加(Δc=Δa±Δb，相消时相对误差暴涨)；乘除→相对误差相加
    (δc=δa+δb)；幂 a^p→δc=p·δa。换算(同一数)：Δ=δ×|x|。"相对×值=绝对"是换算、非传播，别混。
- bf16 与 fp32 同指数位(范围同)、尾数更短(精度差)；fp16 范围小、精度居中。
- "整数的规格化" = 把【整数值】存进【浮点】(1.fffff×2^e 特例)，不是 int 类型本身；int/uint
    类型用补码/纯二进制存、无规格化。同值 13 存 int 与 float 是两种不同位模式。
- 三种编码：float(sign+exp+frac，有指数尾数) / uint(纯二进制大小，无符号) / int(补码，MSB 权重 −2^(N-1))。
- 浮点【恒有符号】(范围靠指数，省 sign 无益却丢负数)；只有整数分 int/uint(省 sign 翻倍正范围且非负量常见)。
- 补码取负 = 逐位取反 +1，因 ~x+1 = 2^N−x ≡ −x (mod 2^N)；好处：加减统一、0 唯一。
    例 int8 −13 = 11110011 = −128+64+32+16+2+1。
- 有效精度=有效数字位数(尾数定)，≠小数点后位数(指数定)；十进制有效位 d≈-log₁₀ε=m×0.301
    (fp32~7/fp16~3/bf16~2)。ε 是相对精度(常数)；绝对步长=ε×数值(随量级变)。
- RoPE cache 三崩点(都汇入 angle=pos×inv_freq → cos/sin)：
    ① positions 大整数【表示不下】(步长≥1 量化；bf16 257 起、fp16 2049 起)→ fp32 让整数精确。
    ② inv_freq【相对精度不足】(非表示不下)：base=10000 bf16 舍成 9984；幂结果跨 4 量级仅 ~2 位
       有效 → fp32 让频率准。
    ③【最致命】相对误差×大 position=巨大绝对角度误差：乘积相对误差相加→angle 带 ε；绝对误差
       =ε×angle，bf16 0.8%×4095≈33 弧度 >> 2π → cos/sin 随机。要 << 2π 须相对误差 ≤1.5e-3，
       唯 fp32(1e-7)够。
  ⟹ 中间量(arange/base^x/cos/sin)恒 fp32(过程需绝对精度)，仅结果 cos/sin∈[-1,1] cast 到 dtype。
- 混合精度(AMP)需 fp32 的算子＝命中三机制之一(见十点五)：①累加/归约(GEMM 累加器、Norm 统计、
    softmax 分母、loss、grad)②超越/大范围(exp、log、sin/cos、logits)③小加大被吞(主权重更新、
    Adam m/v、EMA)。其余 bf16/fp16；默认低精度、敏感处升 fp32、算完降回。
