================================================================================
RoPE（旋转位置编码）算法详解
================================================================================


一、背景与动机
--------------------------------------------------------------------------------

Transformer 模型本身对序列中 token 的顺序无感知（self-attention 是置换不变的），
必须额外引入位置编码才能让模型区分"第 1 个词"和"第 5 个词"。

常见位置编码方案对比：

  方案                代表工作              缺点
  ------------------  --------------------  ------------------------------------
  绝对正弦编码        原始 Transformer      外推性差，无法建模相对关系
  可学习绝对编码      BERT、GPT-2           长度固定，泛化到更长序列困难
  相对位置编码        T5, Transformer-XL   增加额外偏置项，实现复杂
  旋转位置编码 RoPE   LLaMA, Qwen, PaLM    本文重点

RoPE 由苏剑林（Su et al., 2021）在论文
"RoFormer: Enhanced Transformer with Rotary Position Embedding"
（arXiv:2104.09864）中提出。


二、核心目标
--------------------------------------------------------------------------------

设 Query 向量 q 在位置 m，Key 向量 k 在位置 n，
我们希望设计一个位置编码函数 f，使得：

    <f(q, m), f(k, n)>  =  g(q, k, m-n)

即内积结果只依赖相对距离 m-n，而不依赖绝对位置 m 和 n。

这正是 RoPE 的设计目标——用旋转矩阵注入位置信息，
同时保持注意力分数只感知相对位置。


三、二维情形的数学推导
--------------------------------------------------------------------------------

3.1 从复数视角出发

将二维向量 (x1, x2) 视作复数 z = x1 + i·x2。

对位置 m 处的向量施加旋转角 m*theta：

    f(z, m) = z · e^(i·m·theta)
            = (x1 + i·x2)(cos(m*theta) + i·sin(m*theta))
            = (x1*cos(m*theta) - x2*sin(m*theta))
            + i(x1*sin(m*theta) + x2*cos(m*theta))

3.2 验证相对位置性质

    <f(q, m), f(k, n)>
    = Re[ f(q,m) · conj(f(k,n)) ]
    = Re[ q·e^(i*m*theta) · conj(k·e^(i*n*theta)) ]
    = Re[ q · conj(k) · e^(i*(m-n)*theta) ]
    = g(q, k, m-n)   ✓

内积结果仅依赖 m-n，目标成立。


四、高维推广：分组旋转
--------------------------------------------------------------------------------

实际 head dimension 为 d（通常 64、128 等偶数），
将维度两两配对，分成 d/2 个独立的二维子空间，
每个子空间使用不同频率 theta_i。

4.1 旋转角频率定义

对第 i 对维度（i = 0, 1, …, d/2 - 1）：

    theta_i = base^(-2i/d)      （base 通常取 10000）

4.2 旋转矩阵形式

完整的旋转矩阵 R(m) 为分块对角矩阵：

    R(m) =
    [ cos(m*theta_0)  -sin(m*theta_0)                                      ]
    [ sin(m*theta_0)   cos(m*theta_0)                                      ]
    [                                  cos(m*theta_1)  -sin(m*theta_1)     ]
    [                                  sin(m*theta_1)   cos(m*theta_1)     ]
    [                                                          ...          ]
    [                                            cos(m*theta_{d/2-1})  ... ]

编码后的 Query / Key：

    q_tilde = R(m) · q
    k_tilde = R(n) · k

注意力分数：

    score(m, n) = q_tilde^T · k_tilde
                = q^T · R(m)^T · R(n) · k
                = q^T · R(n-m) · k
    （因为 R 是正交矩阵，R^T(m)·R(n) = R(n-m)）

4.3 逐元素高效计算（无需构造大矩阵）

设 q = [q0, q1, q2, q3, …, q_{d-2}, q_{d-1}]，相邻两维配对：

    q_tilde_{2i}   = q_{2i}   * cos(m*theta_i)  -  q_{2i+1} * sin(m*theta_i)
    q_tilde_{2i+1} = q_{2i}   * sin(m*theta_i)  +  q_{2i+1} * cos(m*theta_i)

向量化写法（LLaMA 实现常用）：

    cos_vec     = [cos(m*theta_0), cos(m*theta_0), cos(m*theta_1), ...]   shape: [d]
    sin_vec     = [sin(m*theta_0), sin(m*theta_0), sin(m*theta_1), ...]   shape: [d]
    rotate_half = [-q1, q0, -q3, q2, …]    （奇偶交换并取反）

    q_tilde = q * cos_vec  +  rotate_half(q) * sin_vec

Key 向量 k 同理，位置换成 n 即可。


五、频率设计：theta_i 的物理含义
--------------------------------------------------------------------------------

    theta_i = 10000^(-2i/d)

  维度对索引 i    theta_i 值       含义
  --------------  ---------------  ------------------------------------------
  0（低 i）       theta_0 ≈ 1      旋转快，对位置变化敏感，编码近距离精细关系
  d/2-1（高 i）   theta ≈ 1/10000  旋转慢，对位置变化不敏感，编码远距离粗粒度关系

这与傅里叶多尺度分析思想一致：
  - 低维度对：高频旋转 → 精细编码近距离相对位置
  - 高维度对：低频旋转 → 粗粒度编码远距离相对位置


六、在大模型中的位置：应用于哪一层？
--------------------------------------------------------------------------------

    输入 Token Embeddings
            |
            v
    +-----------------------------------------------+
    |          Transformer Layer × N                |
    |                                               |
    |   +---------------------------------------+   |
    |   |          Self-Attention               |   |
    |   |                                       |   |
    |   |   X  -->  Linear  -->  Q, K, V        |   |
    |   |                        |   |          |   |
    |   |               RoPE ----+   |  <== 此处|   |
    |   |            编码 Q 和 K     |          |   |
    |   |                        v   v          |   |
    |   |        score = (Q_tilde · K_tilde^T) / sqrt(d)  |   |
    |   |        attn  = softmax(score) · V     |   |
    |   +---------------------------------------+   |
    |                    |                          |
    |              Add & LayerNorm                  |
    |              FFN (MLP)                        |
    |              Add & LayerNorm                  |
    +-----------------------------------------------+
            |
            v
         输出 Logits

关键点：
  1. RoPE 只作用于 Q 和 K，不作用于 V
  2. 在每个 Transformer block 的 Multi-Head Self-Attention 中注入
  3. 时机：QKV projection 之后、attention score 计算之前
  4. 每一层的每个注意力头都独立执行 RoPE（共享同一套 theta_i 参数）


七、RoPE 的作用与优势
--------------------------------------------------------------------------------

7.1 核心作用

  作用              说明
  ----------------  ----------------------------------------------------------
  注入位置信息      使模型感知 token 在序列中的顺序
  建模相对位置      注意力分数天然只依赖位置差 m-n，无绝对位置泄漏
  长度外推          训练长度之外的位置仍可计算，支持上下文窗口扩展

7.2 对比绝对位置编码

  绝对编码：
    score(m,n) = (q + PE_m)^T (k + PE_n)
    展开后含 q^T*PE_n、PE_m^T*k 等交叉项 → 隐式编码相对位置，不纯粹

  RoPE：
    score(m,n) = q^T R(n-m) k
    显式且干净地只含相对位置 (n-m)，无冗余项

7.3 上下文长度扩展技术

通过修改 base 或对 theta_i 进行缩放，可扩展有效上下文窗口：

  技术               原理
  -----------------  ---------------------------------------------------------
  Linear Scaling     将位置 m 替换为 m/s（s = 扩展倍数），旋转整体变慢
  NTK-aware Scaling  修改 base -> base * s^(d/(d-2))，统一扩展所有频率
  YaRN               对不同频率的 theta_i 分组处理：低频组插值，高频组保持原样

这些方法均基于 RoPE 的旋转结构，是 LLaMA-2-long、Qwen-long、ChatGLM
等超长上下文模型的核心技术。


八、与其他位置编码的完整对比
--------------------------------------------------------------------------------

  方案                  相对位置    外推能力  计算开销  代表模型
  --------------------  ----------  --------  --------  ----------------------
  Sinusoidal（绝对）    隐式        差        O(1)      原版 Transformer
  Learned PE（绝对）    无          很差      O(1)      BERT, GPT-2
  Relative PE（T5）     显式        中        O(n^2)    T5
  ALiBi                 显式线性    好        O(1)      BLOOM
  RoPE                  显式旋转    好        O(n*d)    LLaMA, PaLM, Qwen


九、完整数学公式汇总
--------------------------------------------------------------------------------

    定义
    -------------------------
    d        : head dimension（偶数）
    base     : 频率基数，默认 10000
    m, n     : token 位置索引（0-based）
    theta_i  : base^(-2i/d),   i = 0, 1, …, d/2 - 1

    编码
    -------------------------
    对位置 m 的向量 q 的第 i 对维度：

    [ q_tilde_{2i}   ]   [ cos(m*theta_i)  -sin(m*theta_i) ] [ q_{2i}   ]
    [ q_tilde_{2i+1} ] = [ sin(m*theta_i)   cos(m*theta_i) ] [ q_{2i+1} ]

    Key 向量 k 同理，使用位置 n。

    注意力分数
    -------------------------
    score(m, n) = q_tilde^T · k_tilde / sqrt(d)
                = q^T · R(n-m) · k / sqrt(d)

    其中 R(n-m) = R(n)^T · R(m) 是只依赖相对位置的旋转矩阵。
