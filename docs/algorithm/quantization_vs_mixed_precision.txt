================================================================================
量化(quantization) vs 混合精度(mixed precision / AMP)：不是一回事
================================================================================

一句话：
  混合精度 = 仍在【浮点】内部混用 fp32/bf16/fp16（非均匀刻度、有 fp32 兜底）；
  量化     = 把浮点换成【低位整数 + 缩放因子 scale】这套新数制（均匀刻度、无 fp32）。
  二者都为省显存/提速而降位数，但换的"东西"不同；fp8 介于两者之间；可串联使用。

关联：
  docs/algorithm/float_binary_representation.txt（浮点三段结构、ε/步长、为何 RoPE/AMP 需 fp32）
  其中第十点五节：大模型里哪些算子需 fp32（混合精度三机制）


================================================================================
一、核心区别：用的还是不是浮点
================================================================================
                混合精度(mixed precision/AMP)        量化(quantization)
  数制          仍是【浮点】fp32/bf16/fp16            换成【整数】int8/int4 + scale（或低位浮点 fp8）
  表示          保留 sign/exp/mantissa，非均匀步长     整数按 scale 映射回实数，均匀步长
  公式          value = ±1.fffff × 2^e                real ≈ scale × (int − zero_point)
  刻度          非均匀（步长 = ε×值，大数粗小数细）     均匀（步长 = scale，全范围一样粗）
  误差性质      相对精度恒定 ε                         绝对精度恒定 = scale（取决于范围/校准）
  目标          训练稳定 + ~2× 提速/省显存             激进压缩(int8 4×、int4 8×) + 整数运算
  fp32 兜底     有（敏感算子升 fp32）                  无 fp32 概念，靠 scale/校准控误差
  主要场景      训练为主（推理也用）                   推理/部署为主（PTQ），也有 QAT


================================================================================
二、混合精度（mixed precision / AMP）
================================================================================
- 全程【浮点】，只是默认 bf16/fp16，敏感算子升 fp32、算完再降回。
- 值仍是"完整范围的浮点数"，误差是浮点的【相对】精度（非均匀：大数步长大、小数步长小）。
- 哪些算子需 fp32（命中三机制之一，详见 float_binary_representation.txt 十点五）：
    A 累加/归约（误差 ~N·ε + 相消）：GEMM 累加器、LayerNorm/RMSNorm 统计、softmax 分母、
      grad all-reduce、loss、grad-norm。
    B 超越/大动态范围：softmax exp、RoPE sin/cos、log/交叉熵、logits。
    C 小量加大量被吞(swamping)：⭐优化器主权重更新 w←w−lr·g、Adam m/v、EMA。
- 典型：AMP 训练（fp32 主权重 + bf16 前向）、bf16 推理。
- 本项目即此类：cos/sin cache 在 fp32 算、cache.to(dtype)；rms_norm_pure 的 x.float()→.to。


================================================================================
三、量化（quantization）
================================================================================
把浮点权重/激活映射成低位【整数】，配一个缩放因子 scale（可含零点 zero_point）：

  量化  ：q = round(x / scale) + zero_point      # x(fp) → 整数 q (int8/int4)
  反量化：x ≈ scale × (q − zero_point)           # 用时还原近似值

------------------------------------------------ (1) 均匀刻度（与浮点的根本不同）
  整数等距 → 实数被映射成 [min, max] 区间里 2^bit 个【等距】档；step = scale，全程一样粗。
  对比浮点【非均匀】(步长=ε×值)：量化是【绝对】精度恒定，浮点是【相对】精度恒定。

------------------------------------------------ (2) 对称 vs 非对称
  对称(symmetric)  ：zero_point=0，范围 [−a, a]，对权重(零均值)常用。
  非对称(asymmetric)：有 zero_point，范围 [min, max]，对激活(如 ReLU 后非负)更省档位。

------------------------------------------------ (3) scale 的粒度（per-X）
  per-tensor：整张量一个 scale（最省、最糙）。
  per-channel/per-group：每通道/每组一个 scale（更准，LLM 权重常用）。
  粒度越细越准，但元数据越多。

------------------------------------------------ (4) 校准与离群值(outlier)
  选 scale 要校准：太大→分辨率浪费；太小→大值溢出截断。
  ⚠ 离群值会撑大范围、毁掉分辨率（均匀刻度的代价：一个大值把 scale 撑大，所有小值全被拖累）。
  LLM 激活有大 outlier → 用 per-channel/per-group、或 SmoothQuant/AWQ 等专门处理 outlier。

------------------------------------------------ (5) 常见档位（LLM）
  INT8 (W8A8)        ：权重+激活都 int8，4× 压缩，整数 GEMM。
  INT4 weight-only (W4A16)：权重 int4、激活 fp16，8× 权重压缩；GPTQ / AWQ，LLM 部署最常用。
  更低：int3/int2、二值等（研究/极端压缩）。

------------------------------------------------ (6) PTQ vs QAT
  PTQ(训练后量化)：训练完直接转 + 少量校准数据定 scale；快、无需重训，精度略降。
  QAT(量化感知训练)：训练时就模拟量化误差(fake-quant)，让模型适应；精度更高、但要重训。


================================================================================
四、和浮点（float_binary_representation.txt）的联系
================================================================================
- 浮点 = 非均匀刻度：相对精度恒定 ε，步长 = ε×值 → 大数粗、小数细。
- 整数量化 = 均匀刻度：绝对精度恒定 = scale，整个 [min,max] 一样粗。
  ⟹ 量化怕 outlier：一个大离群值撑大 scale，所有小值分辨率被拖累（均匀刻度的代价）。
     浮点反而对动态范围友好（指数负责量级），这正是浮点 vs 定点/整数的取舍。
- fp8(E4M3/E5M2)是模糊地带：低位【浮点】(有 exp、非均匀)，但位数已低到接近量化用途 →
  有时归"低精度浮点/混合精度"，有时归"浮点量化"。


================================================================================
五、关系：互补，不互斥
================================================================================
精度/压缩谱：
  fp32 → bf16/fp16 →（fp8）→ int8 → int4 → …
         └── 混合精度的范围 ──┘  └──── 量化的范围 ────┘

- 可串联：先【混合精度训练】(fp32 主权重 + bf16) → 再【量化成 int4 部署】。
- 共同点：都为省显存/提速而降低数值表示的位数。
- 不同点：混合精度换"更短的浮点"+ fp32 兜底；量化换"整数数制"+ scale。


================================================================================
六、速查
================================================================================
- 不是一回事：混合精度=浮点内混 fp32/bf16/fp16(非均匀、有 fp32 兜底，主训练稳定+提速)；
  量化=浮点→低位整数+scale(均匀刻度、无 fp32、怕 outlier，主推理部署的激进压缩)。
- 量化公式：real ≈ scale×(int−zero_point)；对称/非对称；per-tensor/channel/group；PTQ/QAT。
- LLM 常见：W8A8(int8 4×)、W4A16(int4 weight-only 8×，GPTQ/AWQ)。
- 刻度：浮点非均匀(步长=ε×值，相对精度恒定)；量化均匀(步长=scale，绝对精度恒定)→量化怕 outlier。
- fp8 介于两者之间(低位浮点)。二者可串联：混合精度训练 → 量化部署。
- 本项目属混合精度(cache.to(dtype)、rms_norm 的 x.float())，未涉及量化。
